# LANOVEL 서버 기능 명세 (v1.0)

이 문서는 현재 LANOVEL 서버에 구현된 주요 기능들을 요약하여 정리합니다.

## 1. 핵심 기능

- **AI 번역**:
    - Google의 Gemini API (`gemini-2.5-flash` 모델)를 사용하여 일본어 텍스트를 한국어로 번역합니다.
    - API 요청 시 재시도 로직(`tenacity`) 및 오류 처리가 포함되어 안정성을 높였습니다.
    - 특정 문장(예: 광고)을 번역에서 제외하는 기능이 있습니다.

- **데이터베이스 관리**:
    - `SQLite`를 사용하여 모든 데이터를 관리합니다.
    - 데이터베이스 스키마에는 번역 내용, 작품(폴더), 태그, 읽기 활동, 제외 문장 등이 포함됩니다.
    - 서버 시작 시 데이터베이스를 자동으로 초기화하고 필요한 테이블을 생성합니다.

## 2. 웹 인터페이스 (UI)

- **FastAPI 기반 웹 서버**: `FastAPI`를 사용하여 비동기 웹 서버를 구축했습니다.
- **Jinja2 템플릿**: `Jinja2`를 사용하여 다음과 같은 동적 HTML 페이지를 렌더링합니다.
    - `/library`: 전체 작품 목록을 볼 수 있는 서재 페이지.
    - `/book_ui`: 작품 내 URL/챕터 목록을 관리하는 페이지.
    - `/viewer`: 번역된 내용을 읽는 뷰어 페이지.
    - `/admin`: 관리자용 기능(데이터 정리 등)을 위한 페이지.

## 3. 데이터 및 작품 관리

- **작품(폴더) 관리**:
    - 작품을 폴더 단위로 관리합니다.
    - 폴더 생성, 이름 변경 기능을 제공합니다.
    - 특정 작품을 상단에 고정(`pin`)할 수 있습니다.

- **번역 데이터 관리**:
    - 번역된 내용을 추가, 수정, 삭제(CRUD)할 수 있습니다.
    - 여러 번역 데이터를 다른 폴더로 일괄 이동할 수 있습니다.
    - 전체 번역 데이터에 대해 '찾아 바꾸기'를 수행할 수 있습니다.

- **태그 시스템**:
    - 태그를 생성하고 각 작품에 여러 태그를 자유롭게 할당할 수 있습니다. (다대다 관계)

- **읽기 활동 추적**:
    - 작품별로 마지막으로 읽은 위치(`last_read_pid`), 메모, 책갈피 여부를 기록합니다.
    - 작품의 줄거리(summary)를 저장하고 표시할 수 있습니다.

## 4. 고급 및 자동화 기능

- **백그라운드 작업**:
    - `FastAPI`의 `BackgroundTasks`를 활용하여 시간이 오래 걸리는 작업을 백그라운드에서 처리합니다.
    - **제목 자동 가져오기**: 제목이 없는 작품의 URL에 접속하여 `<title>` 태그를 가져와 DB에 업데이트합니다.
    - **줄거리 크롤링**: 지정된 URL에서 작품 줄거리(소개)를 크롤링하여 DB에 저장합니다.

- **자동 데이터베이스 백업**:
    - 서버 누적 실행 시간이 24시간을 초과하면 `backups` 폴더에 자동으로 데이터베이스 파일(`translations.db`)을 백업합니다.

- **PWA (Progressive Web App) 지원**:
    - 오프라인에서도 작동할 수 있도록 서비스 워커(`service-worker.js`)를 제공합니다.
    - 새 버전 배포 시 캐시를 자동으로 갱신하기 위해 현재 Git 커밋 해시를 버전으로 사용합니다.

- **HTML 내보내기**:
    - 특정 작품의 모든 번역 내용을 하나의 독립적인 HTML 파일로 내보내 완전한 오프라인 읽기를 지원합니다.

- **CORS 설정**:
    - 웹 브라우저 확장(Extension) 및 로컬 환경에서의 원활한 API 요청을 위해 CORS(Cross-Origin Resource Sharing)가 설정되어 있습니다.
