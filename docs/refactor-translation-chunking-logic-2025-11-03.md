# 번역 분할(Chunking) 로직 개선

## 변경 이유

페이지당 번역할 텍스트의 양이 적을 경우, 기존의 고정된 분할 방식은 불필요하게 많은 API를 호출하여 비효율을 야기했습니다. 이를 개선하여 API 호출 횟수를 동적으로 조절하고 효율성을 높이고자 합니다.

## 주요 변경 사항

`extension/content.js`의 `translateAllPTags` 함수 내 번역 분할 로직을 다음과 같이 수정했습니다.

- **전체 번역 문장 수**를 기준으로 분할 전략을 동적으로 결정합니다.

1.  **80개 초과 시 (기존 로직 유지):**
    - 기존과 동일하게 약 10개의 묶음(chunk)으로 나누는 공격적인 분할 방식을 사용합니다. (묶음당 5~20개 문장)

2.  **80개 이하 시 (신규 로직 적용):**
    - 더 적은 수의 그룹으로 분할하여 API 호출을 최소화합니다.
    - **20개 문장 이하:** 2개 그룹으로 분할
    - **21개 문장부터:** 10개 문장이 늘어날 때마다 분할 그룹을 1개씩 추가합니다. (예: 30개 문장 -> 3개 그룹, 40개 문장 -> 4개 그룹)

## 기대 효과

- 적은 양의 텍스트 번역 시 API 호출 횟수가 감소하여 번역 속도 및 비용 효율성이 개선됩니다.
- 많은 양의 텍스트 번역 시에는 기존의 안정적인 분할 방식을 유지합니다.
