# 기능 추가: 허용된 출처(Origin) 확인 (v1.0)

## 변경 사항

- **서버 (`server.py`):**
  - CORS 설정에 사용되는 `allow_origins` 목록을 반환하는 `/api/allowed_origins` API 엔드포인트를 새로 추가했습니다.

- **익스텐션 (`content.js`):**
  - 번역 프로세스 시작 전, 현재 페이지의 URL이 서버에서 제공하는 허용된 출처 목록에 포함되어 있는지 확인하는 로직을 추가했습니다.
  - 만약 현재 URL이 허용 목록에 없으면, 익스텐션은 초기화를 중단하고 번역 관련 작업을 수행하지 않습니다. 이로 인해 불필요한 API 요청(`excluded_sentences`, `apply_status` 등)이 발생하지 않습니다.

## 기대 효과

- 의도하지 않은 웹사이트에서 익스텐션이 동작하여 불필요한 리소스를 사용하거나 서버 로그가 남는 것을 방지합니다.
- 번역 기능이 필요한 특정 사이트(예: `syosetu.com`, `kakuyomu.jp`)에서만 익스텐션이 활성화되도록 제한합니다.
