# Fix: CORS 오류 및 URL 매칭 로직 개선 (v1.2)

## 개요

`kakuyomu.jp` 사이트에서 번역 기록이 전혀 인식되지 않는 문제를 해결하고, 전반적인 URL 인식률을 높이기 위해 서버와 클라이언트 코드를 종합적으로 수정했습니다.

## 주요 변경 사항

### 1. CORS (Cross-Origin Resource Sharing) 오류 수정
-   **문제 원인**: `kakuyomu.jp` 도메인이 서버의 CORS 허용 목록에 없어, 해당 사이트에서 서버로 API 요청(번역 기록 조회 등)이 브라우저에 의해 차단되고 있었습니다. 이로 인해 "서버에서 번역 목록을 불러오지 못했다"는 오류가 발생했습니다.
-   **변경 사항 (`server.py`)**: 서버의 `CORSMiddleware` 설정에 `https://kakuyomu.jp`를 `allow_origins` 목록에 추가하여, 해당 도메인에서의 요청을 정상적으로 처리하도록 수정했습니다.

### 2. URL 정규화 및 매칭 로직 강화
-   **URL 프래그먼트(#) 문제 해결 (`content.js`)**:
    -   URL에 포함될 수 있는 `#` 프래그먼트가 DB에 저장된 주소와 불일치를 유발하는 문제를 해결하기 위해, 번역 저장 및 조회 시 `#` 이하를 제거한 "정규화된" URL을 사용하도록 통일했습니다.
-   **URL 매칭 로직 강화 (`server.py`)**:
    -   서버에서 번역 기록을 조회할 때, `http/https`, `www` 서브도메인, 주소 끝의 `/` 등 다양한 URL 변형을 고려하여 조회하도록 매칭 로직을 강화했습니다.
    -   또한, `kakuyomu.jp`의 에피소드 페이지에서 조회 시, 작품의 메인 페이지 URL로 저장된 기록도 함께 찾아오도록 특화 로직을 추가했습니다.

## 기대 효과

-   `kakuyomu.jp` 사이트에서 번역 기능이 정상적으로 동작합니다.
-   과거에 저장된 번역 기록(URL 형식이 다르거나 프래그먼트가 포함된 경우 등)을 다양한 환경에서 안정적으로 불러올 수 있게 됩니다.
