# Fix: 번역 기록 페이지 저장 오류 해결 (v1.0)

## 문제 원인

번역 기록 페이지(`history.html`)에서 개별 항목의 '저장' 버튼을 클릭했을 때, 서버로 전송되는 데이터에 `url`, `original`, `timestamp` 등 필수 필드가 누락되어 있었습니다. 이로 인해 서버의 데이터 유효성 검사에 실패하여 `422 Unprocessable Entity` 오류가 발생하며 저장이 실패했습니다.

## 해결 방안

`history.js`의 `handleHistoryClick` 함수 내 '저장' 버튼 로직을 다음과 같이 수정했습니다.

1.  **전체 데이터 조회**: 수정하려는 항목의 `id`를 사용하여, 페이지 로드 시 저장해 둔 전역 `groupedTranslations` 객체에서 해당 항목의 전체 데이터(url, pid, original 등)를 찾도록 `findTranslationById` 헬퍼 함수를 추가하고 사용했습니다.

2.  **완전한 데이터 객체 생성**: 조회한 원본 데이터에 사용자가 수정한 번역 내용(`translated`)과 새로운 저장 시간(`timestamp`)을 업데이트하여 서버 모델(`TranslationItem`)이 요구하는 모든 필드를 갖춘 완전한 객체를 생성합니다.

3.  **서버 전송**: 이렇게 생성된 완전한 데이터 객체를 `/translations/upsert` 엔드포인트로 전송하여 정상적으로 저장되도록 수정했습니다.

## 기대 효과

-   번역 기록 페이지에서 번역 내용을 수정한 후 '저장' 버튼을 누르면, 변경 사항이 서버에 정상적으로 반영됩니다.
-   데이터가 누락 없이 일관성 있게 관리되어 안정성이 향상됩니다.
