# 폴더 처리 로직 변경 및 개선 v1.5

## 개요

관리자 페이지에서 `null` 폴더(폴더가 지정되지 않은 항목)가 표시되지 않거나 중복으로 표시되는 등 지속적으로 발생하던 문제를 해결하기 위해, 관련 처리 방식을 전면 변경하고 단순화했습니다.

## 핵심 변경 사항

- **'폴더 추가 대기 상태' 개념 도입**:
  - 기존에 '미분류', '폴더 없음' 등 여러 이름으로 혼용되던 `null` 폴더의 표시 이름을 **'폴더 추가 대기 상태'** 로 통일했습니다.

- **서버 로직 일원화 (`server.py`)**:
  - 폴더 목록을 제공하는 API(`get_folders`)와 폴더별 데이터를 그룹화하는 로직(`get_translations_by_folder`) 모두에서 `null` 폴더를 '폴더 추가 대기 상태'로 처리하도록 수정하여 데이터 일관성을 확보했습니다.

- **클라이언트 로직 단순화 (`admin.js`)**:
  - 서버에서 일관된 데이터를 보내줌에 따라, 이름 불일치를 처리하기 위해 추가되었던 복잡한 클라이언트 사이드 로직을 모두 제거했습니다.
  - '폴더로 이동' 드롭다운 메뉴의 옵션 텍스트도 '폴더 추가 대기 상태'로 변경하여 UI 일관성을 맞췄습니다.

## 기대 효과

- `null` 폴더가 항상 목록에 정확히 한 번만 표시됩니다.
- 신규 폴더 생성 시 목록이 즉시 올바르게 갱신됩니다.
- 복잡한 예외 처리 로직이 제거되어 코드의 안정성과 유지보수성이 향상되었습니다.
