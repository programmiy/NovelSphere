# 기능: 뷰어 성능 개선 및 챕터 단위 로딩 도입 (v2.0)

## 개요

대용량 책을 열 때 발생하던 심각한 성능 저하 문제를 해결하고, 사용자의 세부 요구사항에 맞춰 뷰어의 데이터 로딩 방식과 UI를 전면 재구축했습니다.

## 주요 변경 사항

### 1. 챕터 단위 데이터 로딩 및 페이징

- **기존:** 뷰어 진입 시 책에 포함된 모든 번역 기록을 한 번에 불러왔습니다.
- **변경:** 이제 뷰어는 '챕터(동일한 제목을 가진 번역 기록의 묶음)' 단위로 동작합니다.
    1. 뷰어 진입 시, 책 전체의 **챕터 목록**만 먼저 불러옵니다.
    2. 첫 번째 챕터(또는 마지막으로 읽던 챕터)의 내용 전체를 불러옵니다.
    3. 해당 챕터의 내용이 많을 경우, 이전과 같이 **화면 크기에 맞춰 여러 페이지로 자동 분할**하여 보여줍니다.
- **기대 효과:** 초기 로딩 속도가 매우 빠르며, 메모리 사용량이 크게 감소했습니다. 한 번에 한 챕터의 내용만 처리하므로 대용량 책도 원활하게 볼 수 있습니다.

### 2. 챕터 간 동적 탐색

- 현재 읽고 있는 챕터의 마지막 페이지에서 '다음' 버튼(또는 우측 화면 클릭/키보드 →)을 누르면, 다음 챕터의 데이터를 서버에서 새로 불러와 표시합니다.
- 이전 챕터로 돌아가는 경우에도 동일한 방식으로 동작합니다.
- 챕터가 전환될 때, 이전 챕터의 데이터는 메모리에서 즉시 해제하여 효율성을 높였습니다.

### 3. 챕터 기반 목차 사이드바

- 뷰어 헤더의 '목차' 버튼을 통해 열리는 사이드바가 이제 개별 번역 기록(`pid`)이 아닌 **챕터 제목** 목록을 표시합니다.
- 사용자는 목차에서 원하는 챕터를 클릭하여 해당 챕터의 첫 페이지로 즉시 이동할 수 있습니다.

## 수정한 파일

- `server/server.py`: 기존에 잘못 추가했던 API를 삭제하고, 챕터 목록과 특정 챕터의 전체 내용을 제공하는 신규 API 2개를 구현했습니다.
- `server/templates/viewer.html`: 변경 없음 (구조는 이전과 동일)
- `server/static/viewer.css`: 변경 없음 (스타일은 이전과 동일)
- `server/static/viewer.js`: 데이터 로딩, 페이징, 탐색, 목차 로직 전체를 새로운 '챕터 단위' 아키텍처에 맞게 완전히 재작성했습니다.
