# Fix: 사이드바 표시 및 번역 저장 오류 해결 (v1.0)

## 문제 원인

1.  **사이드바 미표시**: `content.js`의 `init` 함수에서 저장된 번역을 적용한 후, `return` 문을 통해 조기 종료되어 사이드바 생성 코드가 실행되지 않았습니다.
2.  **번역 저장 실패**: 개별 번역을 저장하는 `saveSingleTranslation` 함수 내에 정의되지 않은 `loadTranslationsForUrl` 함수를 호출하여 '저장' 버튼이 동작하지 않았습니다.

## 해결 방안

1.  **`init` 함수 로직 개선**:
    *   `applySavedTranslations` 함수가 UI 조작 대신 번역 데이터 객체를 반환하도록 수정했습니다.
    *   `init` 함수는 항상 사이드바를 먼저 주입하고, 반환된 데이터로 내용을 채우도록 변경했습니다.
    *   조기 `return`을 제거하여, 저장된 번역 적용 후에도 새로운 콘텐츠 스캔이 가능하도록 했습니다.

2.  **`saveSingleTranslation` 함수 수정**:
    *   정의되지 않은 함수 호출을 제거했습니다.
    *   `saveAllTranslations`와 동일한 로직을 사용하도록, 개별 번역 데이터를 서버의 `/translations/upsert` 엔드포인트로 전송하여 저장하도록 수정했습니다.

## 기대 효과

-   저장된 번역이 있는 페이지에서도 사이드바가 항상 올바르게 표시됩니다.
-   사이드바에서 '저장' 버튼을 클릭했을 때, 수정된 번역 내용이 서버에 정상적으로 저장됩니다.
-   코드의 안정성과 데이터 일관성이 향상되었습니다.
