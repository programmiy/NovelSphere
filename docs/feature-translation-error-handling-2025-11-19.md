# 자동 번역 오류 처리 및 요청 제한 로직

## 개요
자동 번역 프로세스의 안정성을 높이고 API의 RPM(분당 요청 수) 소모를 관리하기 위해, 특정 조건 하에 번역 요청을 제어하는 새로운 로직이 도입되었습니다.

## 주요 기능 및 변경 사항

### 1. '경계 시간' 기반 자동 중지 (오후 5시)

**목적:** API 사용량 리셋 시간(오후 5시 추정) 전에, 장시간 실행되던 자동 번역 프로세스를 자동으로 중지시켜 일일 사용량을 분석할 시간을 확보합니다.

**동작 방식:**
- **오후 5시 이전에 시작된** 자동 번역 프로세스가 계속 실행되다가, 오후 5시를 넘어서 다음 번역을 요청하는 경우, 해당 요청은 **1회에 한해 거부**됩니다.
- 거부 메시지: `"오후 5시가 되어 자동 번역을 중지합니다. 새로고침 후 다시 시도해주세요."`
- 이 거부 동작 이후, 내부 상태가 초기화됩니다. 따라서 사용자가 **새로고침 하거나 새로 번역을 시작**하는 경우, 오후 5시가 넘었더라도 정상적으로 번역이 실행됩니다.
- 즉, 이 기능은 **오후 5시 이후에 새로 시작된 번역은 막지 않으며**, 오직 5시 이전에 시작되어 계속 실행되던 **자동 번역 프로세스만**을 중지 대상으로 합니다.

### 2. API 과부하(503) 에러 처리 (RPM 소모 방지)

**목적:** 사용자의 귀중한 API RPM 할당량을 보호하기 위해, 과부하 에러 발생 시 불필요한 재시도를 방지합니다.

**동작 방식:**
- 번역 요청 시 API 서버로부터 **503 Service Unavailable (과부하) 에러**가 응답으로 오면, 기존의 재시도 로직을 수행하지 않고 **즉시 요청을 실패 처리**합니다.
- 실패 처리와 동시에 사용자에게 **데스크톱 알림**을 보내 상황을 알립니다.
    - 알림 제목: `번역 API 과부하`
    - 알림 내용: `API 과부하(503)로 요청이 실패했습니다. RPM 한도를 확인하거나 잠시 후 다시 시도하세요.`
- 이를 통해, 과부하 상황에서 재시도가 반복되며 RPM을 빠르게 소진하는 문제를 방지합니다.

### 3. 기타 오류 처리
- 503 이외의 다른 일시적인 HTTP 오류(예: 500 등)에 대해서는 기존과 같이 최대 5회까지 재시도 로직을 수행하여 안정성을 유지합니다.

## 기술적 구현
- `server/routers/translation.py` 파일이 수정되었습니다.
- RPM 소모 방지를 위해, 503 에러 발생 시 재시도 루프를 즉시 중단하는 로직이 추가되었습니다.
- 데스크톱 알림을 위해 `win10toast-persist` 라이브러리가 새로 추가되었으며, `requirements.txt`에 반영되었습니다.
