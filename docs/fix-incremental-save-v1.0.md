# Fix: 번역 중단 시 기록 손실 문제 해결 (v1.0)

## 설명

번역 작업이 진행되는 도중에 사용자가 중지하거나 페이지를 이탈할 경우, 그 시점까지 성공적으로 번역된 기록이 저장되지 않고 손실되는 문제를 해결했습니다.

## 변경 사항

- **저장 방식 변경 (`content.js`):
  -   기존에는 모든 번역 작업이 완료된 후 한 번에 모아서 저장했지만, 이제는 번역된 각 묶음(chunk) 또는 문장이 성공할 때마다 즉시 서버에 저장하도록 변경했습니다.
  -   `translateChunk` 함수와 `retryWithSentenceSplitting` 함수 내에서 번역 성공 직후 `saveAllTranslations`를 호출하여 데이터를 즉시 서버로 전송합니다.
  -   모든 작업 완료 후 마지막에 한 번만 저장하던 로직과 불필요해진 임시 배열(`allFinishedResults`)을 제거했습니다.

## 기대 효과

-   번역 작업이 중간에 중단되더라도, 중단 직전까지 완료된 모든 번역 기록이 안전하게 보존됩니다.
-   네트워크 오류나 사용자의 갑작스러운 페이지 이탈 등 예기치 않은 상황에서도 데이터 손실을 최소화하여 안정성을 높였습니다.
