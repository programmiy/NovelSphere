# Fix: 번역 시 인용부호 누락 문제 해결 (강화판) (v1.0)

## 문제 원인

번역 시 원문에 있던 일본식 인용부호(`「」`, `『』`)가 번역문에 포함되지 않는 문제가 있었습니다. 근본적인 원인은 서버의 AI 프롬프트에 관련 지침이 누락되었기 때문입니다.

## 해결 방안

AI가 지침을 따르지 않을 가능성까지 고려하여, 다음과 같이 이중으로 문제를 해결했습니다.

1.  **프롬프트 수정 (1차 방어)**:
    *   `server.py`에서 Gemini API로 보내는 프롬프트에, 원문의 인용부호를 번역문에도 반드시 보존하라는 명시적인 지침을 추가했습니다.

2.  **코드 후처리 로직 추가 (2차 방어)**:
    *   `background.js`의 `handleBatchTranslate` 함수에 후처리 로직을 추가했습니다.
    *   이 로직은 AI가 만약 지침을 따르지 않고 일반 따옴표(`"..."`)로 응답했을 경우, 원본 텍스트를 확인하여 `「」` 또는 `『』`로 다시 변환해주는 역할을 합니다.

## 기대 효과

-   프롬프트와 코드의 이중 안전장치를 통해, 어떤 경우에도 일본어 인용부호가 번역문에 일관되게 보존되어 번역 품질과 신뢰성이 크게 향상됩니다.
