# feature-site-specific-auto-navigation-2026-02-17

## 변경 사항 요약

- **배경**: 번역 완료 후 다음 페이지로 자동 이동하는 기능이 모든 웹사이트에서 동일하게 `PageDown` 키를 시뮬레이션하도록 동작하여, 특정 웹사이트의 고유한 페이지 넘김 방식과 충돌하거나 비정상적으로 작동했습니다. 특히 `novel18.syosetu.com`과 같은 사이트에서는 `PageDown` 키가 아닌 페이지 내의 '次へ' 버튼을 통해 다음 챕터로 이동해야 하는 요구사항이 있었습니다.
- **변경 내용**: `extension/content.js` 파일의 `translateAllPTags` 함수 내 `startAutoScroll` 로직을 수정하여, 번역 완료 후 다음 페이지로 자동 이동하는 방식을 현재 웹사이트 URL에 따라 다르게 적용하도록 변경했습니다.
    -   `novel18.syosetu.com`: 페이지 끝까지 스크롤한 후, 페이지 내에 존재하는 `<a class="c-pager__item c-pager__item--next">次へ</a>` 요소를 찾아 해당 `href`로 직접 이동하도록 구현했습니다.
    -   `kakuyomu.jp`: 페이지 끝까지 스크롤한 후, 기존과 동일하게 `PageDown` 키를 시뮬레이션하여 다음 페이지로 이동하도록 구현했습니다.
    -   그 외의 모든 웹사이트: `PageDown` 키를 시뮬레이션하는 기본 동작을 유지합니다.
- **목적**: 각 웹사이트의 특성에 맞는 페이지 자동 넘김 방식을 제공하여 사용자 경험을 향상시키고, 번역 자동화의 효율성을 높이는 것을 목표로 합니다.

## 영향

- `novel18.syosetu.com`에서 번역 완료 후 '次へ' 링크를 통해 다음 챕터로 정확하게 이동할 수 있게 됩니다.
- `kakuyomu.jp`에서는 기존과 동일하게 `PageDown` 키 동작으로 다음 페이지로 이동합니다.
- 특정 웹사이트의 페이지 넘김 방식에 맞는 자동화가 가능해져 사용 편의성이 증대됩니다.
- 지원되지 않는 사이트에서는 기존 `PageDown` 키 시뮬레이션 동작을 유지합니다.
