# 번역 저장 시 강제 폴더 지정 문제 수정

## 작업 일시
- 2025년 11월 5일

## 문제 상황
- 라우터 도입 리팩토링 이후, 새로운 번역을 저장할 때 해당 번역의 URL과 동일한 URL을 가진 기존 번역이 폴더에 속해 있으면, 새 번역이 해당 폴더로 강제 지정되는 문제가 발생했습니다.
- 이전에는 이런 경우 폴더가 지정되지 않고 '폴더 추가 대기 상태'가 되는 것이 정상 동작이었습니다.

## 원인 분석
- `server/routers/data_crud.py`의 `upsert_translations` 함수 내에, 새로 추가되는 번역 항목의 `folderName`이 `None`일 경우, 동일 URL을 가진 기존 항목의 폴더를 자동으로 가져와 설정하는 로직이 원인이었습니다.

## 해결 내용
- `upsert_translations` 함수에서 위에서 언급된 자동 폴더 할당 로직을 제거했습니다.
- 이제 새로운 번역 항목은 요청 시 `folderName`이 명시적으로 제공될 때만 폴더에 할당되며, 그렇지 않으면 `folderName`이 `NULL`로 설정되어 '폴더 추가 대기 상태'가 됩니다.

## 기대 효과
- 번역 저장 시 폴더 할당 로직이 이전과 동일하게 동작하여 사용자의 의도치 않은 폴더 지정을 방지합니다.
- 사용자가 직접 새 번역의 폴더를 제어할 수 있게 됩니다.
