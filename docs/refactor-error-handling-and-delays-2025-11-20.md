# 오류 처리 및 지연 시간 리팩토링 (2025-11-20)

## 주요 변경 사항

Gemini API의 과부하(Overload) 및 요청 속도 제한(RPM) 오류에 더 효과적으로 대응하기 위해 `extension/content.js`의 오류 처리 및 지연 시간 로직을 대대적으로 리팩토링했습니다.

### 1. 지수 백오프(Exponential Backoff) 기반 재시도 로직 도입

-   **변경 전**: 통신 오류 발생 시 **60초의 고정된 시간** 동안 대기한 후 재시도했습니다.
-   **변경 후**: **지수 백오프 + Jitter** 전략을 도입했습니다. 오류 발생 시 `(2^시도횟수)초 + 랜덤값` 만큼 대기하여 재시도합니다. 이는 서버 부하를 줄이고 재시도 성공률을 높이는 표준적인 방법입니다.
-   최대 재시도 횟수를 2회에서 5회로 늘려, 일시적인 네트워크 문제나 API 과부하에 대한 대응력을 강화했습니다.

### 2. 묶음(Chunk) 실패 시 재시도 지연 시간 조정

-   **변경 전**: 번역 묶음이 실패하여 더 작은 단위로 나뉘어 재시도될 때, 각 하위 묶음 사이에 **60초**를 대기했습니다.
-   **변경 후**: 해당 대기 시간을 **30초**로 줄였습니다. 묶음 크기를 줄이는 것만으로도 문제가 해결될 수 있으므로, 불필요한 대기 시간을 줄여 효율성을 높였습니다.

### 3. 오류 없는 경우의 묶음 간 대기 시간 단축

-   **변경 전**: 오류 발생 여부와 관계없이 모든 번역 묶음 사이에 **60초**의 고정 대기 시간이 있었습니다.
-   **변경 후**: 이 대기 시간을 **2초**로 대폭 단축했습니다. 이제 오류 처리는 지수 백오프 메커니즘이 전담하므로, 정상적인 상황에서는 더 빠르고 원활한 번역이 가능해졌습니다.
-   긴 텍스트 처리 시의 대기 시간도 일관성을 위해 2초로 조정되었습니다.

## 기대 효과

-   API 과부하(429, 503 오류) 발생 시 더 지능적이고 유연하게 대처하여 번역 성공률을 높입니다.
-   오류가 없는 정상적인 상황에서는 번역 작업이 훨씬 빠르게 완료됩니다.
-   전반적인 번역 프로세스의 안정성과 사용자 경험이 개선됩니다.
