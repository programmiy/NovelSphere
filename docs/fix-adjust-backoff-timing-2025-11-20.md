# 지수 백오프 재시도 대기 시간 조정 (2025-11-20)

## 변경 사항

`extension/content.js`의 지수 백오프 재시도 로직에서, 초기 재시도 대기 시간을 다음과 같이 조정했습니다.

-   **변경 전**: 약 1초부터 시작 (`(2^시도횟수)초`)
-   **변경 후**: **최소 40초**부터 시작하도록 기본 대기 시간을 추가 (`(40 + 2^시도횟수)초`)

## 변경 이유

기존 지수 백오프 방식의 초기 재시도 간격이 너무 짧아, 여러 번역 묶음이 동시에 오류를 일으킬 경우 순간적으로 RPM(분당 요청 수)이 급증하는 문제가 관찰되었습니다.

Gemini API의 지속적인 과부하 상태를 고려하여, 초기 재시도 요청이 서버에 부담을 주지 않도록 최소 대기 시간을 40초로 설정했습니다. 이를 통해 API의 안정성을 최우선으로 확보하고, 재시도 성공률을 높이고자 합니다.
