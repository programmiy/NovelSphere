# 버그 수정: 알림(Toast) 표시 오류 해결 (v1.0)

## 변경 사항 요약
화면 우측 상단에 표시되는 알림(Toast)이 화면 밖으로 벗어나 내용이 보이지 않던 CSS 및 애니메이션 관련 버그를 수정했습니다.

## 문제 원인
`history.html`의 CSS 스타일시트에는 알림을 화면 오른쪽 밖으로 이동시키는 `transform: translateX(100%)` 규칙이 정의되어 있었습니다. 이는 알림이 나타날 때 부드럽게 미끄러져 들어오는 애니메이션 효과를 주기 위함입니다. 하지만 `extension/history.js`의 `showToast` 함수에는 이 `transform` 값을 `translateX(0)`으로 변경하여 알림을 화면 안으로 가져오는 애니메이션 로직이 누락되어 있었습니다. 이로 인해 알림이 생성된 직후 화면 밖에 거의 그대로 머물러 있어, 사용자에게는 일부 그림자나 경계선만 보이는 문제가 발생했습니다.

## 상세 수정 내용

### `extension/history.js`
-   **`showToast` 함수 로직 개선**:
    -   알림 요소가 생성되고 DOM에 추가된 직후, `setTimeout`을 사용하여 `toast.style.transform = 'translateX(0)'`을 실행하도록 변경했습니다. 이 코드는 알림이 부드럽게 화면 안으로 들어오는 애니메이션을 트리거합니다.
    -   3초 후, 다시 `transform` 값을 `translateX(100%)`로 변경하여 알림이 화면 밖으로 사라지게 하고, 애니메이션이 끝난 뒤 DOM에서 요소를 완전히 제거하도록 로직을 추가했습니다.
    -   또한, 알림의 종류(success, error, info)에 따라 배경색을 지정하기 위해 인라인 스타일을 사용하던 방식에서 벗어나, `toast.className = `toast ${type}``과 같이 클래스를 직접 부여하는 방식으로 변경했습니다. 이는 `history.html`에 이미 정의된 CSS 클래스(`.toast.error`, `.toast.info`)를 활용하여 코드의 일관성과 유지보수성을 높입니다.

## 기대 효과
-   **정상적인 알림 표시**: 이제 모든 알림이 화면 우측 상단에 부드럽게 나타났다가, 몇 초 후 사라지는 애니메이션과 함께 정상적으로 표시됩니다.
-   **명확한 피드백 제공**: 사용자는 자신의 작업(저장, 삭제, 오류 등)에 대한 결과를 명확한 텍스트와 색상으로 구분된 알림을 통해 즉각적으로 인지할 수 있게 되어 전반적인 사용성이 향상됩니다.
