# RPM 제한 문제 해결을 위한 번역 대기 시간 증가 (2025-11-13)

## 문제

번역 요청이 너무 많아 API의 분당 요청 수(RPM) 제한에 도달하는 문제가 발생했습니다. 이로 인해 번역 작업이 오히려 지연되는 현상이 있었습니다.

## 해결 방안

번역 요청 묶음(chunk) 사이의 대기 시간을 늘려 RPM을 안정적으로 관리하도록 수정했습니다.

### 주요 변경 사항

1.  **긴 문장 번역 대기 시간 증가:**
    *   150자가 넘는 긴 문장은 개별적으로 번역됩니다.
    *   각 긴 문장 번역 요청 사이의 대기 시간을 기존 2초에서 **10초**로 늘렸습니다.

2.  **일반 문장 묶음 대기 시간 증가:**
    *   일반 문장을 묶어서 번역할 때, 각 묶음(chunk) 사이의 대기 시간을 기존 10초에서 **30초**로 늘렸습니다.

## 기대 효과

- 분당 API 요청 수를 크게 줄여 RPM 제한에 도달하는 것을 방지합니다.
- API 제한으로 인한 작업 실패 및 재시도를 줄여 보다 안정적인 번역 환경을 제공합니다.
