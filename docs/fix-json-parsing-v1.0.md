### 주요 수정사항

*   `handleBatchTranslate` 함수 내에 텍스트 전처리 로직을 추가했습니다.
*   API로 텍스트를 보내기 전, 긴 반복 의성어와 특수 문자(…) 등을 표준화합니다.
*   비정상적인 텍스트 입력으로 인해 API가 유효하지 않은 JSON을 반환하는 문제를 해결합니다.
*   `Failed to parse JSON response from API` 오류의 근본 원인을 해결합니다.
*   입력값을 사전에 제어하여 번역 안정성을 향상시킵니다.

### 보존할 세부 문서

**문제 원인:**
`なあぁぁぁぁぁぁっぁぁぁぁぁぁぁぁぁぁぁぁぁ` 와 같이 매우 길고 특이한 패턴의 텍스트가 API로 전달될 경우, 언어 모델이 응답을 생성하는 과정에서 줄바꿈 등 유효하지 않은 제어 문자를 포함한 JSON 문자열을 반환할 수 있습니다. 이 경우 클라이언트의 `JSON.parse()`가 실패하여 `SyntaxError: Bad control character` 오류가 발생합니다.

**해결 전략:**
방어적으로 API의 응답을 받은 후에 파싱 오류를 처리하는 대신, 문제가 발생할 만한 입력을 사전에 정리하여 API가 안정적인 결과물을 반환하도록 유도하는 것이 더 근본적인 해결책입니다. `handleBatchTranslate` 함수 시작 부분에서 `items.map`을 통해 각 텍스트를 순회하며 정규식을 사용하여 문제가 될 만한 패턴을 제거 또는 치환합니다. 이는 이전에 논의했던 "입력값 제어" 전략의 구체적인 구현입니다.
