# Fix: 번역 안정성 및 오류 처리 로직 개선 (v1.0)

## 설명

API의 `PROHIBITED_CONTENT` 오류 처리 시 확장 프로그램이 멈추거나 충돌하는 문제, 특정 사이트에서의 CORS(Cross-Origin Resource Sharing) 오류, 번역 중단 시 데이터가 손실되는 문제 등 안정성과 관련된 여러 이슈를 해결했습니다.

## 주요 변경 사항

1.  **오류 발생 시 멈춤 현상 해결 (`background.js`, `content.js`):
    -   `background.js`가 오류를 처리할 때, `throw` 대신 구조화된 오류 응답(`sendResponse({ error: '...' })`)을 보내도록 수정하여, 확장 프로그램 컨텍스트가 무효화되며 충돌하던 문제를 해결했습니다.
    -   `content.js`는 이 새로운 오류 응답을 안정적으로 수신하여 재시도 및 분할 로직을 정상적으로 수행합니다.

2.  **`PROHIBITED_CONTENT` 오류 처리 로직 개선 (`server.py`, `content.js`):
    -   `server.py`가 Gemini API의 새로운 오류 형식(`promptFeedback`)을 명시적으로 감지하고, 일관된 오류 메시지를 생성하도록 수정했습니다.
    -   `content.js`는 `PROHIBITED_CONTENT` 오류 수신 시, 작업을 중단하는 대신 다른 통신 오류처럼 묶음을 분할하여 재시도하도록 변경하여 번역 성공률을 높였습니다.

3.  **번역 기록 저장 로직 개선 (`content.js`):
    -   번역 작업이 모두 끝난 후 한 번에 저장하던 방식에서, 번역된 묶음(chunk)이 성공할 때마다 즉시 저장하는 방식으로 변경했습니다.
    -   이를 통해 번역 과정이 중간에 중단되어도 데이터가 손실되지 않도록 안정성을 강화했습니다.

4.  **CORS 오류 수정 (`server.py`):
    -   로컬 서버의 CORS 허용 출처 목록에 `https://novel18.syosetu.com`를 추가하여, 해당 사이트에서도 번역 기록이 정상적으로 저장되도록 수정했습니다.

## 기대 효과

-   API 오류, 특히 콘텐츠 차단 오류 발생 시에도 확장 프로그램이 멈추거나 충돌하지 않고 안정적으로 동작합니다.
-   오류 발생 시 분할 재시도 로직이 정상적으로 작동하여 전체 번역 성공률이 향상됩니다.
-   번역 작업이 중단되어도 진행된 내용이 손실되지 않고 안전하게 보존됩니다.
