# 버그 수정: PID 자연 정렬(Natural Sort) 로직 개선 (v1.0)

## 변경 사항 요약
번역 기록 페이지에서 번역 항목들이 PID(문단 식별자) 순서대로 올바르게 정렬되지 않던 문제를 해결했습니다. 기존의 단순 숫자 변환 방식 대신, 문자와 숫자가 조합된 복잡한 PID도 올바르게 처리할 수 있는 자연 정렬(Natural Sort) 알고리즘을 도입했습니다.

## 문제 원인
기존의 정렬 로직은 PID에서 첫 글자 'p'를 제거하고 나머지를 정수로 변환(`parseInt(pid.replace('p', ''))`)하는 방식이었습니다. 이 방식은 `p1`, `p10`과 같은 단순한 형식의 PID는 처리할 수 있었지만, `la10`과 같이 다른 접두사를 갖거나, 숫자와 문자가 섞인 복잡한 형식의 PID에 대해서는 `NaN`(Not a Number)을 반환하여 정렬 순서를 보장할 수 없었습니다.

## 상세 수정 내용

### `extension/history.js`
-   **`renderGroupedHistory` 함수 내 정렬 로직 변경**:
    -   번역 항목 배열(`filteredTranslations`)을 정렬하는 비교 함수(comparator)를 자연 정렬 알고리즘으로 교체했습니다.
    -   새로운 알고리즘은 정규식을 사용해 PID 문자열을 문자 부분과 숫자 부분으로 분리합니다.
    -   분리된 각 부분을 순서대로 비교하되, 숫자 부분은 수의 크기로, 문자 부분은 사전 순으로 비교합니다.
    -   이를 통해 `p1`, `p2`, `p10`, `la1`, `la2`, `la10` 등 다양한 형식의 PID가 사용자의 직관에 맞게 올바르게 정렬됩니다. (예: `p2`가 `p10`보다 앞에 오게 됩니다.)

## 기대 효과
-   **정확한 정렬**: 모든 번역 기록이 PID 순서에 따라 일관되고 정확하게 정렬되어, 사용자가 원하는 순서대로 번역 내용을 확인할 수 있습니다.
-   **안정성 및 확장성 확보**: 향후 `p`나 `la` 외에 다른 형식의 PID가 추가되더라도, 별도의 코드 수정 없이 안정적으로 정렬 로직이 작동할 수 있는 확장성을 확보했습니다.
