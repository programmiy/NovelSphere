# Fix: 번역 기록/사이드바 버그 수정 (v1.1)

## 문제 원인

여러 UI 및 데이터 관련 버그가 보고되었습니다.
1.  **폴더 전체 선택 오류:** '전체 선택' 체크박스 클릭 시 폴더가 접히고, 하위 항목 체크가 보이지 않았습니다.
2.  **대량 작업 불가:** 상단 대량 작업 버튼들이 비활성화되어 동작하지 않았습니다.
3.  **항목 정렬 오류:** URL 그룹 내 번역 아이템이 `pid` 순서가 아닌, 서버에서 받은 순서(최신순)로 표시되었습니다.
4.  **데이터 중복 생성:** 사이드바에서 번역 수정 시, 폴더 정보가 누락되어 별도의 그룹으로 표시되는 문제가 있었습니다.

## 해결 방안

`server.py`와 `history.js` 파일을 다음과 같이 수정했습니다.

1.  **폴더 정보 유실 해결 (`server.py`):**
    *   `/translations/upsert` 엔드포인트 로직을 수정했습니다.
    *   **수정 시:** 폴더 정보가 없는 요청의 경우, DB에 저장된 기존 폴더 값을 그대로 유지하도록 변경했습니다.
    *   **추가 시:** 폴더 정보가 없을 경우, 같은 URL의 다른 항목이 속한 폴더를 자동으로 상속받도록 변경했습니다.

2.  **UI 및 정렬 문제 해결 (`history.js`):**
    *   **강제 접힘 방지:** '전체 선택' 체크박스 영역 클릭 시에는 폴더가 접히지 않도록 `handleHistoryClick` 함수에 예외 조건을 추가했습니다.
    *   **대량 작업 버튼 활성화:** 아이템 선택 시 대량 작업 버튼이 활성화되도록 `updateBulkActionPanel` 함수 로직을 수정했습니다.
    *   **`pid` 정렬:** (코드 재확인 결과) `pid` 숫자 순 정렬 기능은 이미 올바르게 구현되어 있었습니다. 강제 접힘 문제가 해결됨에 따라 정상적으로 동작할 것으로 예상됩니다.

## 기대 효과

-   사이드바에서 번역을 수정해도 더 이상 중복 그룹이 생성되지 않습니다.
-   '폴더 전체 선택' 기능이 폴더를 접지 않고 정상적으로 동작하며, 이에 따라 대량 작업(삭제, 이동)도 원활하게 사용할 수 있습니다.
-   번역 기록이 `pid` 순서로 올바르게 표시됩니다.
