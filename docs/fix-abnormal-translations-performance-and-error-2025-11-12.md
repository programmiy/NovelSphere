# 버그 수정: '이상번역 찾기' 기능 오류 및 성능 문제 해결

## 작업 일자

2025-11-12

## 문제 상황

'이상번역 찾기' 기능과 관련하여 다음과 같은 복합적인 문제가 보고되었습니다.

1.  **초기 보고**: 버튼을 클릭해도 아무런 반응이 없음.
2.  **추가 정보**: 기능이 매우 느리게 동작하며, 브라우저가 몇 초간 멈추는 현상 발생 (`[Violation] 'click' handler took...` 메시지).
3.  **추가 오류**: 서버 측에서 `TypeError`가 발생하는 등 안정성 문제 발생.
4.  **정확도 문제**: 기능이 일본어 문자가 아닌 일반 문장부호(예: `「`, `」`, `・`, `ー`)까지 '이상 번역'으로 잘못 감지함.

## 문제 원인 및 해결 과정

이 문제는 클라이언트와 서버 양단에 걸친 복합적인 성능 병목 및 필터링 로직의 정확도 문제였습니다.

### 1차 진단: 클라이언트 측 필터링

-   **원인**: 브라우저가 모든 번역 데이터를 받아와 자바스크립트로 필터링하는 과정에서, 데이터가 많을 경우 브라우저가 멈추는 것으로 판단.
-   **해결 시도**: 필터링 로직을 서버로 이전하기로 결정하고, `GET /translations/abnormal` API를 신설.

### 2차 진단: 서버 측 필터링

-   **원인**: 1차 해결 시도 후에도 여전히 지연 문제가 발생. 새로 만든 서버 API가 DB의 모든 레코드를 메모리로 불러온 후, 파이썬 코드로 필터링하면서 서버에 병목이 발생.
-   **해결**: 데이터베이스의 성능을 최대한 활용하도록 구조를 다시 변경.
    1.  **SQLite에 REGEXP 함수 등록 (`database.py`)**: SQLAlchemy 이벤트 리스너를 사용하여 SQLite에 커스텀 정규식 함수를 등록.
    2.  **DB 쿼리 최적화 (`data_crud.py`)**: `REGEXP` 함수를 포함한 SQL 쿼리를 실행하여 DB 레벨에서 직접 '이상 번역' 레코드를 찾아오도록 수정.
    3.  **`TypeError` 해결 (`data_crud.py`)**: DB 조회 결과 객체를 `_asdict()` 메서드를 통해 올바르게 `dict`로 변환하도록 수정.

### 3차 진단: 클라이언트 측 렌더링

-   **원인**: 서버 응답이 빨라진 후에도, 찾아낸 '이상 번역'의 수가 많을 경우(`수천 ~ 수만 건`) 브라우저가 그 결과를 화면에 그리는 과정에서 다시 멈추는 현상 확인. 이는 DOM 요소를 반복적으로 추가하면서 발생하는 렌더링 부하 때문.
-   **해결**: `renderGroupedHistory` 함수(`admin.js`)의 렌더링 방식을 최적화.
    1.  **`DocumentFragment` 사용**: 모든 폴더 그룹 DOM 요소를 `DocumentFragment`에 먼저 추가.
    2.  **단일 DOM 추가**: 루프가 끝난 후, 완성된 `DocumentFragment`를 실제 DOM에 단 한 번만 추가. 이로써 DOM 조작 횟수를 최소화하여 렌더링 성능을 대폭 향상.

### 4차 진단: 필터링 정확도

-   **원인**: 일본어 문자를 감지하는 정규식(`JAPANESE_REGEX_PATTERN`)의 범위가 너무 넓어, 한국어 번역에서도 사용될 수 있는 CJK 공통 문장부호(`「`, `」`, `。` 등)까지 일본어로 잘못 판단하는 문제 발견.
-   **해결**: `data_crud.py`의 정규식 패턴을 수정하여, 순수 일본어 문자(히라가나, 가타카나, 한자)의 유니코드 범위만 포함하도록 수정. 특히 `・` (가타카나 중간점)와 `ー` (장음 부호)를 제외하여 정확도를 높임.

## 최종 결과

-   **성능**: 서버(DB 쿼리)와 클라이언트(DOM 렌더링) 양단의 성능을 모두 최적화하여, 데이터 양에 관계없이 '이상번역 찾기' 기능이 즉각적으로 반응하고 브라우저 멈춤 현상이 완전히 해결되었습니다.
-   **정확성**: 필터링 로직의 정규식을 수정하여, 불필요한 문장부호(`・`, `ー` 포함)를 제외하고 실제 일본어 문자가 포함된 번역만 정확하게 감지하도록 개선되었습니다.
-   **안정성**: 무거운 연산을 데이터베이스 엔진에 위임하고, 서버와 클라이언트의 오류 처리 및 렌더링 로직을 개선하여 안정성을 크게 높였습니다.
-   **구조 개선**: 클라이언트-서버-데이터베이스 각 계층이 자신의 역할에 맞는 최적의 작업을 수행하도록 아키텍처를 개선했습니다.
