# 번역 로직 수정 및 안정성 강화

## 작업 일시
- 2025년 11월 5일

## 문제 상황
- 리팩토링 이후 번역 기능이 일본어 원문을 그대로 반환하는 문제가 지속적으로 발생했습니다.
- 초기 진단으로 API 응답 파싱 로직을 수정했으나 문제가 해결되지 않아 추가 조사를 진행했습니다.

## 원인 분석
1.  **복잡한 프롬프트:** Gemini API로 전송되는 프롬프트의 지시사항이 너무 복잡하여, `response_mime_type="application/json"` 설정과 충돌을 일으키거나 모델의 오작동을 유발할 가능성이 있었습니다.
2.  **불안정한 파싱 로직:** 기존의 정규식 기반 JSON 파싱 로직은 API 응답의 사소한 변화에도 취약하여 파싱 실패 시 원문을 그대로 반환하게 만드는 근본적인 원인이었습니다.

## 해결 내용
1.  **프롬프트 단순화:** API로 보내는 프롬프트를 "Translate the text..."로 시작하는 간결하고 명확한 지시로 변경하여 모델의 혼동 가능성을 줄였습니다.
2.  **파싱 로직 교체:** 불안정한 정규식 파싱 코드를 완전히 제거하고, `json.loads()`를 사용하는 표준적이고 안정적인 방식으로 교체했습니다. 또한, 파싱 실패 시 명확한 오류를 반환하도록 예외 처리를 강화했습니다.

## 기대 효과
- 번역 기능의 안정성과 신뢰성이 크게 향상됩니다.
- 향후 유사한 문제가 발생할 가능성을 최소화하고, 오류 발생 시 원인 추적이 용이해집니다.
