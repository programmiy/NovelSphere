# Fix: 파서 및 금지된 콘텐츠 오류 처리 개선 (v1.0)

## 설명

Gemini API의 민감한 콘텐츠 필터링으로 인해 발생하는 `PROHIBITED_CONTENT` 오류와, 번역된 텍스트에 큰따옴표가 포함될 때 발생할 수 있는 JSON 파싱 오류를 동시에 해결하기 위해 서버 로직을 수정했습니다.

## 변경 사항

1.  **API 프롬프트 수정 (`server/server.py`):
    -   API로 보내는 프롬프트의 예시에서 이스케이프된 큰따옴표(`\"`)를 제거하고 단순한 JSON 형태로 변경했습니다.
    -   이를 통해 API의 콘텐츠 필터가 덜 민감하게 반응하도록 유도하여 `PROHIBITED_CONTENT` 오류 발생 가능성을 줄였습니다.

2.  **JSON 파싱 로직 강화 (`server/server.py`):
    -   API 응답을 파싱하는 로직을 더 견고하게 변경했습니다.
    -   기존의 단일 정규식 대신, 각 JSON 객체를 개별적으로 찾고 그 안에서 `id`와 `translated` 필드를 추출하는 방식을 사용합니다.
    -   이 새로운 방식은 모델이 반환한 `translated` 텍스트 내에 이스케이프 처리되지 않은 큰따옴표가 포함되어 있어도 구문 오류 없이 안정적으로 파싱할 수 있습니다.

3.  **명시적인 오류 처리 (`server/server.py`):
    -   콘텐츠 필터링(`PROHIBITED_CONTENT`) 등 API가 비정상적인 이유로 번역을 거부했을 때, 서버가 원인을 명확하게 포함한 오류 메시지를 반환하도록 처리 로직을 추가했습니다.

## 기대 효과

- 불필요한 콘텐츠 검열 발생 빈도 감소.
- 번역 결과물에 큰따옴표가 포함되더라도 안정적으로 파싱하여 데이터 유실 방지.
- 오류 발생 시 원인 파악 용이.
