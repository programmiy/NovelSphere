# 기능 추가: 전체 찾아 바꾸기 (v1.0)

## 변경 사항 요약
UI 개선 및 서버 이관 과정에서 누락되었던 '전체 찾아 바꾸기' 기능을 복구하고 재구현했습니다. 이제 사용자는 번역 기록 페이지 상단의 버튼을 통해 모든 번역 기록에 대해 특정 단어를 일괄적으로 찾아 다른 단어로 교체할 수 있습니다.

## 상세 내용

### 1. 백엔드 (`server/server.py`)
-   **`POST /translations/find_replace` 엔드포인트 추가**:
    -   `find_text`와 `replace_text`를 포함하는 요청을 받아 처리하는 새로운 엔드포인트를 구현했습니다.
    -   이 엔드포인트는 데이터베이스의 `translations` 테이블에 있는 모든 레코드의 `translated` 열에 대해 SQLite의 `REPLACE()` 함수를 사용하여 일괄적으로 내용을 교체합니다.
    -   작업이 완료되면, 몇 개의 항목이 업데이트되었는지 결과 메시지를 반환합니다.

### 2. 프론트엔드 (`extension/history.html`)
-   **찾아 바꾸기 모달 HTML 추가**:
    -   '찾을 내용'과 '바꿀 내용'을 입력받는 폼과 '취소', '실행' 버튼을 포함하는 모달 창의 HTML 코드를 추가했습니다.
    -   이 모달은 기존의 '번역 수정' 모달과 동일한 CSS 스타일을 공유하여 일관된 UI를 제공합니다.

### 3. 프론트엔드 (`extension/history.js`)
-   **이벤트 리스너 및 핸들러 함수 추가**:
    -   상단의 '전체 찾아 바꾸기' 버튼(`openGlobalFindReplaceBtn`)에 대한 클릭 이벤트 리스너를 추가하여, 클릭 시 모달 창을 열도록 했습니다.
    -   모달 창의 닫기, 취소, 실행 버튼에 대한 이벤트 리스너를 각각 추가했습니다.
    -   `executeGlobalFindReplace` 함수를 구현하여, '실행' 버튼 클릭 시 모달의 입력 값을 가져와 사용자에게 최종 확인을 받은 후, 백엔드의 `/translations/find_replace` 엔드포인트로 API 요청을 보냅니다.
    -   요청이 성공적으로 완료되면, 서버로부터 받은 결과 메시지를 토스트 알림으로 사용자에게 보여주고, `loadAndRenderGrouped()` 함수를 호출하여 변경된 내용이 반영된 번역 기록을 다시 불러옵니다.

## 기대 효과
-   **기능 복구**: 서버 이관 중 누락되었던 핵심 편의 기능인 '전체 찾아 바꾸기'를 복구하여 사용자의 작업 효율성을 높입니다.
-   **사용성 향상**: 자주 사용되는 용어나 특정 단어의 번역을 한 번에 수정할 수 있어, 수동으로 여러 항목을 수정해야 하는 번거로움을 크게 줄여줍니다.
