# 기능: URL 적용 기능 및 관리자 UI 오류 수정 (v1.0)

## 요약

이번 업데이트에서는 관리자 페이지의 사용성을 개선하고 안정성을 높이는 데 중점을 두었습니다. 특정 URL의 번역본을 실제 웹페이지에 적용할지 여부를 선택하는 'URL 적용' 기능을 추가했으며, 이와 함께 관리자 페이지의 여러 UI 오류를 수정했습니다.

## 주요 변경 사항

### 1. URL 적용 기능 추가

- **`applied_urls` 테이블 추가**: 특정 URL의 번역본을 페이지에 자동으로 적용할지 여부를 저장하는 새로운 데이터베이스 테이블을 추가했습니다.
- **적용/해제 API 엔드포인트 추가** (`/api/urls/apply`):
  - 관리자 페이지에서 버튼 클릭으로 특정 URL을 '적용' 상태로 설정하거나 해제할 수 있습니다.
- **상태 확인 API 엔드포인트 추가** (`/api/urls/apply_status`):
  - 콘텐츠 스크립트가 페이지 로드 시 해당 URL의 번역을 적용해야 하는지 서버에 질의할 수 있습니다.
- **관리자 페이지 UI 연동**:
  - 각 URL 그룹 헤더에 '적용'/'해제' 버튼을 추가했습니다.
  - 이 버튼의 상태는 서버에 저장된 `applied_urls` 테이블의 내용과 동기화됩니다.

### 2. 관리자 페이지 UI 오류 수정

- **렌더링 오류 해결**: `admin.html`의 잘못된 HTML 태그 중첩 구조를 수정하여 페이지가 깨지는 문제를 해결했습니다.
- **자바스크립트 오류 수정**: `admin.js`의 `renderGroupedHistory` 함수 내에 있던 심각한 구문 오류를 수정하여, 페이지 전체가 렌더링되지 않던 문제를 해결했습니다.
- **통계 섹션 복구**: 위의 렌더링 및 스크립트 오류가 해결됨에 따라, 정상적으로 데이터를 받아와 통계(총 번역 수 등)를 표시하게 되었습니다.
- **토스트 알림 회귀 문제 해결**:
  - 간헐적으로 토스트 알림이 5%만 표시되던 문제를 해결했습니다.
  - 자바스크립트에서 인라인 스타일을 직접 제어하는 대신, CSS 클래스(`.show`)를 추가/제거하는 방식으로 변경하여 애니메이션 동작의 안정성을 높였습니다.

## 기대 효과

- 사용자는 이제 관리자 페이지에서 특정 소설의 특정 화차 번역본을 페이지에 적용할지 여부를 직접 제어할 수 있습니다.
- 관리자 페이지의 레이아웃 깨짐 및 기능 미작동 문제가 해결되어 안정적인 사용이 가능합니다.
- 토스트 알림이 모든 상황에서 정상적으로 표시됩니다.
