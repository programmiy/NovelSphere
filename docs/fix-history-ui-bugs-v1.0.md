# Fix: 번역 기록 페이지 UI 버그 수정 (v1.0)

## 문제 원인

번역 기록 페이지에서 여러 UI 관련 버그가 발견되었습니다.
1.  **항목 정렬:** URL 그룹 내의 번역 아이템들이 `pid` 순서가 아닌, 서버에서 받은 순서(최신순)로 표시되었습니다.
2.  **중복 버튼:** 각 URL 그룹 헤더에 기능이 중복되는 '찾아 바꾸기' 버튼이 있었습니다.
3.  **대량 작업 불가:** '폴더 전체 선택' 등으로 아이템을 선택해도, 상단의 '선택 삭제' 및 '폴더로 이동' 버튼이 비활성화 상태로 남아있어 사용할 수 없었습니다.

## 해결 방안

`history.js` 파일의 UI 렌더링 및 이벤트 처리 로직을 다음과 같이 수정했습니다.

1.  **`pid` 기준 정렬 추가**: `renderGroupedHistory` 함수 내에, 각 URL 그룹의 아이템들을 화면에 그리기 전에 `pid`를 기준으로 숫자 오름차순 정렬하는 로직을 추가했습니다.
2.  **중복 버튼 제거**: `renderGroupedHistory` 함수의 HTML 템플릿에서 불필요한 '찾아 바꾸기' 버튼 코드를 삭제했습니다.
3.  **대량 작업 버튼 활성화**: `updateBulkActionPanel` 함수를 수정하여, 아이템 선택 여부에 따라 '선택 삭제' 버튼과 '폴더로 이동' 드롭다운의 `disabled` 속성을 동적으로 변경하도록 했습니다.

## 기대 효과

-   번역 기록이 `pid` 순서(1화, 2화, ...)로 올바르게 표시되어 가독성이 향상됩니다.
-   중복 UI가 제거되어 화면이 깔끔해집니다.
-   '전체 선택' 후 대량 삭제 및 폴더 이동 기능이 정상적으로 동작합니다.
