# 기능: 번역 요청 간 최소 딜레이 설정 (v1.0)

## 요약

번역 API에 대한 요청 빈도를 제어하기 위해, 각 번역 요청 묶음(청크) 사이에 최소 10초의 딜레이를 두도록 `content.js`의 로직을 수정했습니다.

## 변경 사항

- `extension/content.js` 파일의 `translateAllPTags` 함수 내에 있는 동적 딜레이 계산 로직을 수정했습니다.
- 기존의 최소 딜레이 값이었던 1000ms (`Math.max(1000, ...)` )를 10000ms로 변경하여, 각 번역 청크 처리 후 최소 10초의 대기 시간을 갖도록 설정했습니다.

## 기대 효과

- 단시간에 너무 많은 요청이 번역 API로 전송되는 것을 방지하여, API 사용량 제한(rate limit) 초과와 관련된 오류 발생 가능성을 줄입니다.
- 보다 안정적인 번역 환경을 제공합니다.
