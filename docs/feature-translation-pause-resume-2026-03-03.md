# 번역 중단 및 재개 기능 추가 (2026-03-03)

## 개요
번역 중 검열(Safety Block)이나 API 오류로 인해 번역이 중단될 경우, 프로세스가 종료되지 않고 확장 프로그램 UI에서 사용자가 직접 확인 및 수정 후 번역을 **재개**할 수 있는 기능을 추가했습니다. 또한 수동 입력 시 즉시 저장할 수 있도록 UI 편의성을 개선했습니다.

## 변경 사항

### 1. 확장 프로그램 (Extension)
- **UI 개선:** 사이드바 헤더에 '번역 재개(▶️)' 버튼을 추가했습니다.
- **실행 컨텍스트 유지:** 서버로부터 `response.stop` 신호를 받으면 `await`를 사용하여 해당 지점에서 실행을 멈춥니다.
- **수동 입력 활성화:** 
    - 번역 대기 중인 항목(placeholder)도 처음부터 편집 가능(`contenteditable="true"`)하도록 설정했습니다.
    - 사용자가 텍스트를 입력하면 즉시 '저장' 버튼이 활성화됩니다.
    - 중단 시 현재 처리 중이던 항목의 "⏳ 번역 중..." 문구를 자동으로 지워 바로 입력할 수 있게 돕습니다.
- **재개 로직:** 사용자가 '재개' 버튼을 클릭하면 멈춰있던 지점부터 다시 번역을 진행합니다.

### 2. 서버 (Server)
- **응답 방식:** 검열 발생 시 `HTTPException(status_code=429)`를 반환하여 확장에 중단 신호를 보냅니다. 불필요한 서버 측 `input()` 대기를 제거하여 안정성을 높였습니다.

## 사용 방법
1. 번역 중 검열 발생 시 사이드바 로그에 안내 메시지가 표시되고 번역이 일시 중지됩니다.
2. 사용자가 중단된 칸에 직접 번역문을 입력합니다. (입력 시 '저장' 버튼이 활성화됨)
3. '저장' 버튼을 눌러 수동 번역을 저장합니다.
4. 사이드바 상단의 ▶️ 버튼을 클릭하여 번역을 재개합니다.
