# Fix: 번역 서버 내부 오류 해결 (v1.0)

## 문제 원인

`background.js`에서 번역을 요청할 때, 로컬 파이썬 서버(`server.py`)에서 처리되지 않은 예외가 발생하여 "Internal Server Error"를 반환했습니다. 이로 인해 `background.js`는 JSON이 아닌 응답을 파싱하려다 `SyntaxError`를 일으켰습니다. 근본적인 원인은 Gemini API로 요청을 보낼 때, 여러 텍스트를 하나의 JSON 문자열로 묶어 프롬프트에 포함시키는 복잡한 방식의 불안정성으로 추정됩니다.

## 해결 방안

`server.py`의 `/translate` 엔드포인트 로직을 다음과 같이 수정했습니다.

1.  **API 요청 방식 변경**:
    *   기존: 번역할 텍스트 목록을 JSON 문자열로 만들어 프롬프트의 일부로 전송했습니다.
    *   변경: 텍스트 목록을 단순 개행(`\n`) 문자로 구분하여 하나의 일반 텍스트로 만든 후, 프롬프트에 포함하여 Gemini API에 전송합니다.

2.  **API 응답 처리 방식 변경**:
    *   Gemini API가 반환한 번역 결과(개행으로 구분된 텍스트)를 `.split('\n')`을 통해 다시 배열로 변환하여 최종 결과를 생성합니다.

## 기대 효과

-   복잡한 JSON 직렬화/역직렬화 과정을 제거하여 API 요청/응답 처리의 안정성을 크게 향상시켰습니다.
-   "Internal Server Error"가 해결되어 번역 기능이 정상적으로 동작합니다.
