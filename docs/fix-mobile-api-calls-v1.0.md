## 모바일 환경 API 호출 문제 해결 (v1.0)

### 배경
모바일 환경에서 관리 페이지 접속 시 "데이터 로딩 실패: 서버가 작동 중인지 확인해주세요"라는 토스트 메시지가 뜨며 데이터 로딩에 실패하는 문제가 발생했습니다. 이는 `server/static/admin.js` 파일 내 JavaScript 코드가 API 요청 시 `http://127.0.0.1:8001`과 같이 서버 주소를 하드코딩하여, 모바일 기기에서 해당 주소로 서버에 접근할 수 없었기 때문입니다.

### 변경 사항
`server/static/admin.js` 파일 내의 모든 `http://127.0.0.1:8001` 문자열을 제거했습니다. 이로써 API 호출 시 상대 경로를 사용하게 되어, 웹 페이지가 로드된 현재 출처(Origin)를 기준으로 API 요청이 이루어지게 됩니다.

```javascript
// 변경 전 예시
fetch(`http://127.0.0.1:8001/translations_by_folder?t=${Date.now()}`)

// 변경 후 예시
fetch(`/translations_by_folder?t=${Date.now()}`)
```

### 영향
*   모바일 기기에서 관리 페이지 접속 시 API 호출이 정상적으로 이루어집니다.
*   데이터 로딩 실패 토스트 메시지가 더 이상 표시되지 않고, 번역 기록 및 기타 데이터가 올바르게 로드됩니다.
*   모바일 환경에서의 애플리케이션 사용성이 크게 향상됩니다.
