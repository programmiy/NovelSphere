### 주요 수정사항

*   `GEMINI.md` 와 `LANOVEL/GEMINI.md` 파일의 작업 절차를 업데이트했습니다.
*   코드 수정과 문서 생성을 하나의 커밋으로 묶도록 명시하여, 브랜치 병합 시 발생하던 충돌을 방지합니다.
*   기존의 모호했던 지침을 더 명확하고 실행 가능한 단계별 워크플로우로 개선했습니다.

---

### 보존할 세부 문서

**문제 원인:**
기존 작업 절차는 코드 변경 커밋과 문서 생성 커밋을 분리하여, 브랜치 전환 시 문서 파일이 추적되지 않은 상태로 남아 병합 충돌을 일으켰습니다.

**해결 전략:**
사용자의 제안에 따라, 코드와 관련 문서를 항상 원자적 단위로 취급하는 새로운 워크플로우를 채택하고 이를 `GEMINI.md`에 명시했습니다. 모든 관련 변경사항(코드, 문서)을 하나의 커밋에 포함시킴으로써 Git 히토리를 깔끔하게 유지하고, 브랜치 작업의 안정성을 높입니다.
