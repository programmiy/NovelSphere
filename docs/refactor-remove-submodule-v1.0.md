### 주요 수정사항

*   프로젝트 구조를 단순화하기 위해 `LANOVEL` 서브모듈을 완전히 제거했습니다.
*   Git 내부 설정 손상으로 인해 표준 명령이 동작하지 않아, 수동으로 서브모듈의 흔적을 모두 정리했습니다.
*   이제 이 프로젝트는 서브모듈이 없는 단일 저장소로 작동합니다.
*   모든 소스 코드는 루트 디렉토리에 있는 파일들을 기준으로 관리됩니다.

---

### 보존할 세부 문서

**문제 원인:**
`LANOVEL` 서브모듈의 Git 설정(`.gitmodules` 등)이 손상되어, `git submodule` 관련 명령이 모두 실패하는 불안정한 상태였습니다. 또한, 사용자는 서브모듈 구조 자체의 복잡성을 원하지 않았습니다.

**해결 전략:**
표준 명령을 우회하여 수동으로 서브모듈을 정리했습니다.
1.  `.gitmodules` 설정 파일 삭제 (이미 존재하지 않았음)
2.  `git rm --cached LANOVEL` 명령으로 Git 추적 목록에서 서브모듈을 제거
3.  `rmdir` 명령으로 실제 서브모듈 디렉토리를 물리적으로 삭제

이 과정을 통해 저장소를 깨끗한 단일 저장소 구조로 리팩토링하여, 향후의 버전 관리 복잡성을 근본적으로 해결했습니다.
