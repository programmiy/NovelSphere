# 기능: 뷰어 스와이프 모드 UI 개선 (v1.0)

## 개요

모바일 환경의 뷰어에서 '넘김 방식'을 '스와이프' 전용으로 설정했을 때, 더 깔끔한 읽기 환경을 제공하기 위해 UI를 동적으로 변경하고 관련 버그를 수정했습니다.

## 주요 변경 사항

1.  **UI 동적 숨김 처리** (`viewer.css`, `viewer.js`):
    *   사용자가 넘김 방식을 '스와이프'로 설정하면, `<body>` 태그에 `swipe-active` 클래스를 추가합니다.
    *   이 클래스가 활성화되면, CSS를 통해 화면 상단의 헤더(`viewer-header`)와 페이지 넘김을 위한 좌우 터치 영역(`.nav-arrow`)이 숨겨집니다.
    *   다른 모드('터치' 또는 '모두')로 변경하면 `swipe-active` 클래스가 제거되어 UI가 다시 나타납니다.

2.  **버그 수정** (`viewer.js`):
    *   '스와이프' 모드에서도 화면을 터치하면 숨겨졌던 좌우 화살표 UI가 다시 나타나는 버그를 수정했습니다.
    *   마우스 움직임에 반응하는 `handleBookMouseMove` 함수가 '스와이프' 모드에서는 실행되지 않도록 가드 구문을 추가하여 문제를 해결했습니다.
