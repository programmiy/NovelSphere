# fix-translation-retry-delay-2026-02-17

## 변경 사항 요약

- **배경**: 배치 번역이 `PROHIBITED_CONTENT`로 인해 차단된 후 개별 항목 번역으로 전환될 때, API 요청이 너무 빠르게 연속적으로 발생하여 `429 Client Error: Too Many Requests` 오류가 자주 발생했습니다. 이는 전체 번역 프로세스의 지연과 실패의 원인이 되었습니다.
- **변경 내용**: `server/routers/translation.py` 파일 내에서 개별 항목을 번역하는 루프에 각 항목 번역 요청 후에 1초의 지연(`time.sleep(1)`)을 추가했습니다.
- **목적**: 이 변경은 API 요청 빈도를 줄여 `429 Client Error: Too Many Requests` 오류 발생을 최소화하고, 개별 항목 번역의 안정성을 향상시켜 전체 번역 프로세스의 성공률과 효율성을 높이는 것을 목표로 합니다.

## 영향

- `429 Client Error: Too Many Requests` 오류가 감소하여 번역 안정성이 향상됩니다.
- 개별 항목 번역 시 각 항목 사이에 1초의 지연이 추가되므로, 많은 수의 항목을 개별 번역할 경우 전체 완료 시간이 약간 증가할 수 있습니다. 그러나 오류 감소로 인한 재시도 감소로 실제 체감 성능은 개선될 것으로 예상됩니다.
