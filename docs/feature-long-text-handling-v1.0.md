# Feature: 긴 텍스트 개별 처리 기능 (v1.0)

## 기능 설명

번역 요청 시, API의 안정성을 높이고 오류 가능성을 줄이기 위해 원문이 긴 텍스트를 분리하여 처리하는 기능을 추가했습니다.

## 구현 내용

1.  **긴 텍스트 기준 설정**:
    *   `content.js` 상단에 `LONG_TEXT_THRESHOLD` 상수를 추가하여 긴 텍스트를 구분하는 기준을 150자로 설정했습니다.

2.  **`translateAllPTags` 로직 변경**:
    *   번역할 전체 텍스트 목록을 수집한 후, 150자를 초과하는 '긴 텍스트'와 그 외의 '일반 텍스트'로 목록을 분리합니다.
    *   **긴 텍스트 처리**: API 오류를 방지하기 위해 각 항목을 개별적으로, 1초의 간격을 두고 순차적으로 번역 요청합니다.
    *   **일반 텍스트 처리**: 기존과 동일하게 여러 항목을 묶어 배치(batch) 방식으로 번역을 요청하여 효율성을 유지합니다.

## 기대 효과

-   매우 긴 텍스트가 포함된 요청으로 인해 발생할 수 있는 API 오류나 타임아웃 가능성을 줄여 번역 과정의 전반적인 안정성이 향상됩니다.
-   사용자에게 긴 텍스트가 개별 처리되고 있음을 로그로 알려주어 현재 진행 상황에 대한 투명성을 제공합니다.
