# 버그 수정: 알림(Toast) 표시 오류 재발 해결 (v1.1)

## 변경 사항 요약
이전 병합(merge) 과정에서 유실되었던 알림(Toast) 표시 로직을 복원하여, 알림이 화면 밖으로 벗어나 보이지 않던 문제를 다시 해결했습니다.

## 문제 원인
`extra` 브랜치를 `main`으로 병합하는 과정에서 충돌이 발생했으며, 충돌 해결 시 `extension/history.js` 파일의 `showToast` 함수가 이전에 수정했던 버전이 아닌, 애니메이션 로직이 누락된 과거 버전으로 되돌아갔습니다. 이로 인해 알림이 화면에 정상적으로 나타나지 않는 버그가 재발했습니다.

## 상세 수정 내용

### `extension/history.js`
-   **`showToast` 함수 로직 복원**:
    -   이전에 수정했던 것과 동일하게, 알림 요소에 `transform` 스타일을 적용하여 슬라이드 애니메이션이 정상적으로 작동하도록 로직을 다시 적용했습니다.
    -   알림의 종류(success, error, info)에 따라 CSS 클래스를 부여하는 로직도 함께 복원하여, 인라인 스타일 대신 CSS 클래스를 통해 배경색이 올바르게 적용되도록 했습니다.

## 기대 효과
-   **버그 재발 해결**: 모든 알림이 다시 화면 우측 상단에 정상적으로 표시됩니다.
-   **코드 안정성**: 병합 과정에서 발생할 수 있는 코드 유실 문제를 인지하고, 재수정을 통해 코드의 안정성을 확보했습니다.
