# Feature: 빈 번역 찾기 기능 및 저장 로직 개선 (v1.0)

## 개요

번역 관리 페이지의 사용성을 개선하고, 번역 실패 시 데이터 저장 방식을 변경합니다.

## 주요 변경 사항

### 1. '빈 번역 찾기' 기능 추가

-   **`admin.html` / `admin.js`**:
    -   번역 관리 페이지(`admin.html`)에 '빈 번역 찾기' 버튼을 추가했습니다.
    -   이 버튼을 클릭하면 번역 내용이 비어있는(`''`) 항목만 필터링하여 보여줍니다.
    -   검색창에 내용을 입력하면 '빈 번역 찾기' 필터는 자동으로 비활성화됩니다.
    -   반대로 '빈 번역 찾기'를 활성화하면 검색어는 초기화됩니다.

### 2. 번역 실패 시 저장 로직 변경

-   **`extension/content.js`**:
    -   문장 분할 재시도까지 최종적으로 실패한 항목에 대해, 기존에는 페이지 제목을 변경하려던 로직을 제거했습니다.
    -   대신, 번역이 실패한 항목은 이제 번역 내용을 빈 문자열(`''`)로 하여 서버에 저장됩니다.
    -   이로 인해 '빈 번역 찾기' 기능을 통해 실패한 항목들을 쉽게 찾아 관리할 수 있습니다.

### 3. 재시도 로직 수정

-   **`extension/content.js`**:
    -   단일 항목 번역 실패 시, 불필요하게 묶음 단위를 줄이려 시도하던 로직을 수정하여, 실패한 항목이 1개일 경우 즉시 개별 항목 재시도 로직으로 전환되도록 효율성을 개선했습니다.

## 영향

-   **`extension/content.js`**: `retryWithSentenceSplitting` 및 `handleFailure` 함수가 수정되었습니다.
-   **`server/templates/admin.html`**: '빈 번역 찾기' 버튼이 추가되었습니다.
-   **`server/static/admin.js`**: '빈 번역 찾기'를 위한 상태 변수, 이벤트 리스너 및 필터링 로직이 추가/수정되었습니다.
