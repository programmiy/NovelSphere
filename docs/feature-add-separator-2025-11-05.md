# 번역 중단 구분선 추가

## 작업 일시
- 2025년 11월 5일

## 요청 사항
- 사용자가 번역 중단 규칙에 새로운 구분선 `◎◎◎`를 추가해달라고 요청했습니다.

## 해결 내용
- `extension/content.js` 파일의 `translateAllPTags` 함수 내부에 있는 구분선 확인 로직을 수정했습니다.
- 기존의 `----------`와 공백/em dash 외에, `◎◎◎` 문자열이 포함된 문단을 만나면 그 이후의 번역을 중단하도록 조건을 추가했습니다.

## 기대 효과
- 사용자가 정의한 새로운 구분선 `◎◎◎`를 기준으로 번역 범위를 효과적으로 제어할 수 있습니다.
