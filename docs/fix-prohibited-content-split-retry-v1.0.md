# Fix: PROHIBITED_CONTENT 오류 시 분할 재시도 (v1.0)

## 설명

API로부터 `PROHIBITED_CONTENT` 오류를 수신했을 때, 해당 번역 묶음(chunk) 전체를 포기하던 기존 로직을 수정했습니다. 이제부터는 다른 통신 오류와 마찬가지로, 작업을 더 작은 단위로 분할하여 재시도하도록 변경되었습니다.

## 변경 사항

- **오류 처리 로직 수정 (`content.js`):
  -   `translateChunk` 함수의 `catch` 블록에서 `PROHIBITED_CONTENT` 오류를 특별 취급하여 즉시 반환하던 코드를 제거했습니다.
  -   이제 이 오류는 일반적인 통신 오류와 동일한 처리 절차를 따릅니다. 즉, 몇 차례의 재시도 후에도 실패하면 `handleFailure` 함수가 호출되어 해당 묶음을 더 작은 묶음으로 나누어 다시 번역을 시도합니다.

## 기대 효과

-   번역 묶음의 일부 문장만 API의 콘텐츠 필터에 의해 거부되는 경우, 문제가 없는 나머지 문장들은 성공적으로 번역될 수 있습니다.
-   전체 번역 성공률이 향상되고, 불필요하게 번역이 실패하는 경우를 줄여줍니다.
