# Fix: '페이지에 적용' 수동 로직 복원 및 개선 (v1.0)

## 문제 원인

1.  **강제 번역 적용:** 소설 페이지 로드 시, '페이지에 적용' 버튼 클릭 여부와 관계없이 번역이 강제로 적용되었습니다. 이는 과거 사이드바 로딩 버그를 수정하면서, 원래의 '수동 적용' 확인 로직을 제거하고 '자동 적용'으로 변경했기 때문입니다.
2.  **'페이지에 적용' 버튼 미구현:** 번역 기록 페이지의 버튼이 실제 동작 로직과 연결되어 있지 않았습니다.

## 해결 방안

원래의 '수동 적용' 설계를 복원하고 사용자 경험을 개선하기 위해 `content.js`와 `history.js`를 모두 수정했습니다.

1.  **`history.js` 수정 (제어 기능 복원):**
    *   `chrome.storage`에 URL별 '적용 허용' 상태(`status-[url]`)를 저장하고 변경하는 `toggleApplyStatus` 함수를 구현했습니다.
    *   페이지 로드 시 이 상태를 함께 불러와, '페이지에 적용' 버튼에 현재 상태('적용 해제' 등)가 올바르게 표시되도록 수정했습니다.
    *   버튼 클릭 시 `toggleApplyStatus`가 호출되도록 이벤트 핸들러를 연결했습니다.

2.  **`content.js` 수정 (적용 기능 복원):**
    *   `applySavedTranslations` 함수를 수정하여, 페이지 로드 시 `chrome.storage`의 '적용 허용' 상태를 먼저 확인하고, `true`일 때만 번역을 적용하도록 원래 로직을 복원했습니다.
    *   **(개선)** `chrome.storage.onChanged` 리스너를 추가하여, 번역 기록 페이지에서 '적용' 상태를 변경했을 때, 소설 페이지를 새로고침하지 않아도 실시간으로 번역이 적용되거나 해제(새로고침)되도록 개선했습니다.

## 기대 효과

-   사용자가 번역 기록 페이지의 '페이지에 적용' 버튼을 통해 특정 소설 페이지의 번역 적용 여부를 직접 제어할 수 있습니다.
-   '강제 번역 적용' 문제가 해결되어, 원하지 않는 번역이 자동으로 표시되지 않습니다.
-   실시간 리스너 추가로 사용자 편의성이 향상되었습니다.
