# 테스트 자동화 도입

## 내용

- `pytest`와 `TestClient`를 도입하여 핵심 API 및 기능에 대한 단위/통합 테스트 작성 환경을 구축했습니다.

## 세부 작업

- `tests/` 디렉토리 생성하여 테스트 코드 중앙 관리
- 기존 `server/test_sample.py`의 테스트를 `tests/test_main.py`로 통합 및 기존 파일 삭제
- `pytest.ini` 파일을 생성하여 `pythonpath`를 설정함으로써, 테스트 실행 시 모듈을 정상적으로 찾을 수 있도록 수정
- `pytest` 실행을 통해 모든 테스트가 성공적으로 통과하는 것을 확인

## 기대 효과

- 코드 변경에 대한 안정성 확보 및 버그 사전 방지
