# 수정: API 429 오류 시 번역 자동 중지

## 개요

API 서버로부터 429 (Too Many Requests) 오류 코드를 수신했을 때, 번역 프로세스가 자동으로 중지되도록 `background.js`의 오류 처리 로직을 수정했습니다.

## 주요 변경 사항

- **429 오류 특별 처리**:
  - `background.js`의 `onMessage` 리스너에서 발생하는 오류를 확인하여, 상태 코드가 429인 경우를 별도로 처리합니다.
- **자동 중지 메커니즘**:
  - 429 오류가 감지되면, `content.js`로 `{ notification: "...", stop: true }` 형태의 메시지를 전송합니다.
  - `content.js`는 이 메시지를 수신하여, 사용자가 '번역 중지' 버튼을 직접 누른 것과 동일하게 번역 프로세스를 중단하고 관련 UI를 업데이트합니다.
- **사용자 경험 개선**:
  - 과도한 API 요청으로 인한 반복적인 오류 발생과 재시도를 방지하여, 확장 프로그램의 안정성과 사용자 경험을 개선합니다.
