# 번역 기록 페이지 UI/UX 개선 (v1.3)

## 개요
사용자의 피드백에 따라, 페이지의 모든 기능이 항상 표시되도록 UI 가시성을 개선했습니다. 이전 버전에서 숨겨져 있던 항목별 액션 버튼과 일괄 작업 패널의 표시 방식을 변경하여 사용자가 모든 기능을 한눈에 파악하고 접근할 수 있도록 했습니다.

## 주요 변경 사항

### 1. 항목별 액션 버튼 항상 표시
- 각 번역 기록 항목(`history-item`)의 '수정', '복사', '삭제' 버튼이 마우스 오버 시에만 나타나던 것을, 항상 표시되도록 `history.html`의 CSS를 수정했습니다.

### 2. 일괄 작업 패널 가시성 개선
- 일괄 작업 패널(`bulk-actions`)이 항목 선택 시에만 나타나던 것을, 항상 페이지에 표시되도록 변경했습니다.
    - `history.html`에서 `bulk-actions` 패널의 `display: none` 스타일을 제거했습니다.
    - 패널 내의 '폴더로 이동' 드롭다운과 '선택 삭제' 버튼은 초기 상태에서 `disabled` 처리하여 비활성화했습니다.
    - `history.js`의 `updateBulkActionPanel` 함수를 수정하여, 선택된 항목이 있을 때만 해당 컨트롤들을 활성화하고, 선택된 항목이 없을 때는 다시 비활성화하도록 로직을 변경했습니다.
    - 선택된 항목이 없을 때 표시되는 텍스트를 '선택된 항목 없음'으로 명확히 변경했습니다.

## 완료된 개선 사항
- 모든 기능 요소의 가시성 향상
- 새로운 디자인 테마 및 스타일 (Material Design 아이콘, 라이트/다크 모드, 페이드인 애니메이션, 아코디언 애니메이션)
- 레이아웃 구조 변경 (테이블 형태의 목록 뷰)
- 핵심 기능 UI 개선 (컨트롤 패널 재그룹화, 일괄 작업 패널 개선, 로딩 상태 표시)

모든 계획된 UI/UX 개선 사항 및 사용자 피드백이 반영되었습니다.
