# 아코디언 기본 상태 및 알림 기능 복구 (v1.0)

## 개요
사용자 피드백에 따라 아코디언 그룹의 기본 상태를 다시 접힌 상태로 되돌리고, 사라졌던 페이지 상태 알림 기능을 복구했습니다.

## 주요 변경 사항

### 1. 아코디언 그룹 기본 상태 되돌리기
- `history.js` 파일의 `renderView` 함수에서 아코디언 그룹을 렌더링할 때, `list-group-toggle` 아이콘을 `▲`에서 `▼`로 다시 변경했습니다.
- `list-group-content` 요소에 `collapsed` 클래스를 다시 추가하여 페이지 로드 시 번역 기록 그룹이 기본적으로 접힌 상태로 표시되도록 했습니다.

### 2. 페이지 상태 알림 기능 복구
- **HTML 요소 추가:** `history.html`의 `main-header` 바로 아래에 `id="status-message"`를 가진 `div` 요소를 추가했습니다. 이 요소는 페이지 우측 상단에 위치하며, 작업 상태를 사용자에게 알리는 역할을 합니다.
- **CSS 스타일링:** `status-message` 요소에 대한 기본 스타일(배경색, 글자색, 패딩, 둥근 모서리, 폰트 크기, `z-index`, `display: none`으로 초기 숨김)을 인라인 스타일로 적용했습니다.
- **JavaScript 함수 추가:** `history.js`에 `showStatusMessage(message, type = 'info')` 함수를 추가했습니다. 이 함수는 메시지를 받아 `status-message` 요소에 표시하고, 3초 후에 자동으로 숨깁니다.
- **주요 액션에 통합:** 다음 함수들이 실행된 후 `showStatusMessage`를 호출하도록 통합했습니다.
    - `copyTranslation`: "번역이 클립보드에 복사되었습니다."
    - `saveEditedTranslation`: "번역이 성공적으로 저장되었습니다."
    - `deleteSelectedTranslations`: "N개의 번역이 삭제되었습니다."
    - `clearAllTranslations`: "모든 번역 기록이 삭제되었습니다."

## 결과
- 번역 기록 그룹이 페이지 로드 시 기본적으로 접힌 상태로 표시됩니다.
- 주요 사용자 액션 후에 페이지 우측 상단에 해당 작업의 상태를 알려주는 메시지가 나타났다가 사라집니다.

이 수정으로 사용자 의도에 맞는 아코디언 기본 동작이 복구되었고, 사라졌던 알림 기능이 다시 제공되어 사용자 경험이 향상되었습니다.
