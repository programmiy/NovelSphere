# 서버 및 확장 프로그램 버그 수정

**문서 버전:** 1.0
**수정 일자:** 2025-10-09

## 1. 개요

번역 기능 사용 중 발생한 여러 버그를 수정하여 안정성과 사용성을 개선했습니다.

---

## 2. 주요 수정 사항

### 2.1. 서버: API 응답 처리 안정성 강화 (`server/server.py`)

- **문제점**: Gemini API가 가끔씩 따옴표가 누락되는 등 약간 손상된 JSON을 반환했을 때, `json.loads()` 파서가 오류를 발생시키며 번역 프로세스가 실패했습니다.
- **해결책**: 엄격한 JSON 파서 대신, 정규표현식(Regex)을 사용하여 응답에서 `id`와 `translated` 쌍을 직접 추출하는 방식으로 변경했습니다. 이를 통해 사소한 문법 오류가 있어도 프로그램이 중단되지 않고, 올바른 형식의 데이터만 안정적으로 처리하게 됩니다.

### 2.2. 확장 프로그램: API 요청 제한(429) 오류 처리 로직 수정 (`extension/background.js`)

- **문제점**: `429 Too Many Requests` 오류 발생 시, 재시도 및 묶음 분할 로직이 실행되지 않고 번역 작업이 즉시 중단되었습니다.
- **해결책**: `background.js`가 429 오류를 받았을 때, 오류 정보를 `content.js`로 올바르게 전파하도록 수정했습니다. 이로써 `content.js`의 `handleFailure` 함수가 정상적으로 호출되어, 더 작은 단위로 작업을 분할하여 자동으로 재시도하게 됩니다.

### 2.3. 확장 프로그램: 사이드바 UI 가독성 개선 (`extension/content.js`)

- **문제점**: 번역 사이드바 하단의 로그 콘솔에서, 경고/오류 메시지가 일반 텍스트와 같은 색상으로 표시되고 시간과 메시지 사이의 간격이 없어 가독성이 떨어졌습니다.
- **해결책**: `content.js`에 주입되는 CSS 스타일을 수정하여 다음 두 가지를 적용했습니다.
    1.  `.log-error` 클래스에 빨간색(` #ff6b6b`)을 적용하여 오류 메시지가 눈에 띄도록 수정했습니다.
    2.  `.log-time` 클래스에 오른쪽 여백(margin-right)을 추가하여 시간과 메시지 텍스트를 명확히 구분했습니다.
