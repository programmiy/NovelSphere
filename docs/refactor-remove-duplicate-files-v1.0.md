### 주요 수정사항

*   메인 저장소(루트)에 중복으로 존재하던 확장 프로그램 소스 파일들을 모두 삭제했습니다.
*   이제 모든 소스 코드는 `LANOVEL/` 서브모듈 내에서만 관리됩니다.
*   프로젝트 구조를 단순화하여, 서브모듈을 사용하는 본래 의도에 맞게 정리했습니다.
*   이를 통해 향후 Git 작업 시 발생할 수 있는 혼란과 충돌을 방지합니다.

---

### 보존할 세부 문서

**문제 원인:**
메인 저장소와 `LANOVEL` 서브모듈에 동일한 소스 파일들이 중복으로 존재하여 Git 작업 흐름이 복잡해지고 잦은 충돌의 원인이 되었습니다. 이는 의도치 않게 파일이 복제되면서 발생한 구조적 문제였습니다.

**해결 전략:**
사용자의 동의 하에, 메인 저장소에 있는 중복 파일들을 과감히 삭제하는 리팩토링을 진행했습니다. 이제 메인 저장소는 최상위 문서들과 서브모듈의 '컨테이너' 역할만 수행하고, 실제 코드는 서브모듈이 독립적으로 관리하는 명확한 구조가 되었습니다.
