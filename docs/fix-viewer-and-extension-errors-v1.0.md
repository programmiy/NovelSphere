# 뷰어 및 익스텐션 오류 수정 (v1.0)

이 업데이트는 뷰어와 브라우저 익스텐션에서 발생한 여러 오류를 해결합니다.

## 주요 변경 사항

### 1. 뷰어 아이콘 초기화 오류 해결 (`ReferenceError`)

- **문제**: 페이지 로드 시 야간/주간 모드 아이콘(`ICON_MOON` 등)이 초기화되기 전에 참조되어 `ReferenceError`가 발생했습니다. 이로 인해 스크립트 실행이 중단되어 무한 로딩 및 아이콘 깨짐 현상이 나타났습니다.
- **해결**: 아이콘 상수 선언부를 `viewer.js` 스크립트 상단으로 이동하여, 함수 호출 전에 항상 상수가 초기화되도록 수정했습니다.

### 2. Favicon 404 오류 해결

- **문제**: 브라우저가 탭 아이콘(`favicon.ico`)을 요청할 때 파일이 존재하지 않아 서버 로그에 404 오류가 기록되었습니다.
- **해결**: `viewer.html`의 `<head>` 태그에 `<link rel="icon" href="data:,">`를 추가하여, 브라우저가 불필요한 아이콘 요청을 보내지 않도록 처리했습니다.

### 3. 익스텐션 메시지 채널 오류 해결

- **문제**: 페이지가 새로고침될 때 번역 요청이 진행 중인 경우, `content.js`와 `background.js` 간의 통신 채널이 닫혀 콘솔에 `Uncaught (in promise)` 오류가 발생했습니다.
- **해결**: `content.js`의 메시지 전송 로직(`translateChunk`, `retryWithSentenceSplitting`)에 오류 처리 구문을 추가했습니다. 통신 채널이 닫혔을 때 발생하는 특정 오류를 감지하고, 재시도하는 대신 작업을 정상적으로 중단하도록 수정하여 불필요한 오류 메시지가 표시되지 않도록 했습니다.
