### 주요 수정사항

*   `background.js`의 `chrome.runtime.onMessage` 리스너를 수정했습니다.
*   모든 `sendResponse` 호출을 `try...catch` 블록으로 감쌌습니다.
*   비동기 작업 중 메시지 채널이 닫혔을 때 발생하는 콘솔 오류를 방지합니다.
*   이를 통해 사용자가 페이지를 일찍 떠나는 경우에도 오류 로그가 남지 않게 됩니다.
*   코드 안정성을 높여 잠재적인 경쟁 상태를 더 잘 처리합니다.

---

### 보존할 세부 문서

**문제 원인:**
`Chunk translation failed (...) the message channel closed before a response was received` 오류는 비동기 작업(특히 재시도 포함 번역)이 완료되기 전에 사용자가 탭을 닫거나 다른 페이지로 이동할 때 발생합니다. 백그라운드 스크립트가 응답을 보내려 할 때, 응답을 받을 대상(콘텐츠 스크립트)이 이미 사라져 통신 채널이 닫혀있기 때문입니다.

**해결 전략:**
이것은 기능 중단보다는 경쟁 상태에 따른 자연스러운 오류입니다. 해결책은 오류를 막는 것이 아니라, 오류가 발생해도 콘솔에 불필요한 로그를 남기지 않도록 처리하는 것입니다. `sendResponse` 자체를 `try...catch`로 감싸면, 채널이 닫혔을 때의 예외를 조용히 처리하여 콘솔을 깨끗하게 유지하고, 다른 예기치 않은 오류는 여전히 기록할 수 있습니다.
