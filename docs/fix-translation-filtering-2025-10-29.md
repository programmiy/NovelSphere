
# 기능: 번역 필터링 강화 (tagCount 및 구분선)

## 개요

번역 요청 시 불필요한 API 호출을 줄이고, 원치 않는 번역을 방지하기 위해 클라이언트 사이드(브라우저 확장 프로그램)에서 번역 대상을 필터링하는 로직을 강화했습니다.

## 변경 사항

- **`extension/content.js` 수정**:
  - `translateAllPTags` 함수 내에서 번역할 텍스트를 수집하는 로직을 변경했습니다.
  - `pid`가 `tagCount`인 문단은 번역 대상에서 제외하도록 조건을 추가했습니다.
  - 텍스트 내용에 구분선(`----------` 또는 `—` 등)이 포함된 문단은 번역을 시도하지 않고 즉시 건너뛰도록 수정했습니다.
  - 기존에는 구분선을 만나면 그 *이후*의 문단만 번역을 중단했지만, 이제는 구분선 자체도 번역 대상에서 제외됩니다.

## 기대 효과

- `tagCount`와 같이 번역이 필요 없는 정보가 번역 API로 전송되는 것을 방지하여 비용을 절감합니다.
- 구분선이 번역되는 것을 막아 더 깔끔한 번역 결과를 얻을 수 있습니다.
- 클라이언트 단에서 미리 필터링하므로 서버의 부담을 줄여줍니다.
