# PWA (Progressive Web App) 지원 추가

## 개요
이 업데이트는 애플리케이션에 PWA(Progressive Web App) 기능을 추가하여, 사용자가 웹 애플리케이션을 데스크톱이나 모바일 장치에 설치하고 오프라인에서도 사용할 수 있도록 합니다. 이를 통해 사용자 경험을 향상시키고 접근성을 높입니다.

## 변경 사항

### 1. HTML 파일 업데이트
`manifest.json` 링크와 서비스 워커 등록 스크립트가 다음 HTML 템플릿에 추가되었습니다:
*   `server/templates/admin.html`
*   `server/templates/book_ui.html`
*   `server/templates/log_viewer.html`

`server/templates/library.html` 및 `server/templates/viewer.html` 파일은 이미 PWA 관련 설정이 되어 있어 변경되지 않았습니다.

### 2. 아이콘 파일 추가
PWA 매니페스트에서 참조하는 아이콘 파일이 `server/static` 디렉터리에 추가되었습니다:
*   `server/static/icon-192x192.svg`: 제공된 SVG 코드를 사용하여 생성되었습니다.
*   `server/static/icon-512x512.png`: 루트 디렉터리에 있던 `book.png` 파일을 이동 및 이름 변경하여 사용되었습니다.

### 3. `manifest.json` 업데이트
`server/static/manifest.json` 파일이 업데이트되어 새로 추가된 아이콘 파일을 올바르게 참조하도록 수정되었습니다.
*   `icon-192x192.svg`는 `sizes: "any"` 및 `type: "image/svg+xml"`로 설정되었습니다.
*   `icon-512x512.png`는 `sizes: "512x512"` 및 `type: "image/png"`로 설정되었습니다.

### 4. 서버 라우팅 업데이트
`server/server.py` 파일에 `/service-worker` 경로에 대한 리디렉션 라우트가 추가되었습니다. 이는 `.js` 확장자 없이 서비스 워커를 요청하는 경우(예: 일부 브라우저 또는 확장 프로그램) 발생할 수 있는 404 오류를 방지하고, `service-worker.js` 파일이 올바르게 제공되도록 합니다.

### 5. 동적 데이터 사전 캐싱 기능 추가
`library.js` 및 `library.css` 파일이 업데이트되어 모든 책을 한 번에 다운로드할 수 있는 전역 "모든 책 오프라인 다운로드" 버튼이 추가되었습니다. 이 버튼을 클릭하면 모든 책의 챕터 및 번역 데이터를 서비스 워커 캐시에 미리 저장하여 오프라인 상태에서도 상세 내용을 볼 수 있도록 합니다. 다운로드 진행 상황이 버튼 옆에 표시됩니다. 개별 책 카드에 있던 "오프라인 다운로드" 버튼은 제거되었습니다.

## 사용 방법
PWA 기능을 사용하려면, 지원되는 브라우저(예: Chrome)에서 애플리케이션에 접속한 후, 브라우저 주소 표시줄 또는 메뉴에서 "앱 설치" 또는 "홈 화면에 추가" 옵션을 선택하여 설치할 수 있습니다. 설치 후에는 오프라인 상태에서도 애플리케이션의 기본 기능을 사용할 수 있습니다.

새로 추가된 "오프라인 다운로드" 기능을 사용하려면, 서재 페이지 상단 헤더에 있는 "모든 책 오프라인 다운로드" 버튼을 클릭하십시오.

## 검증 방법
1.  브라우저 캐시를 지웁니다.
2.  애플리케이션에 접속합니다.
3.  브라우저 개발자 도구(예: Chrome DevTools)의 'Application' 탭에서 'Manifest' 및 'Service Workers' 섹션을 확인하여 PWA가 올바르게 등록되었는지 확인합니다.
4.  브라우저에서 PWA 설치 프롬프트가 나타나는지 확인하고, 설치를 시도하여 앱이 정상적으로 설치되는지 확인합니다.
5.  서재 페이지에서 "모든 책 오프라인 다운로드" 버튼을 클릭하여 동적 데이터가 캐시되는지 확인합니다.
6.  서버를 종료한 후, PWA를 실행하여 캐시된 동적 데이터(예: 다운로드한 책의 챕터)에 접근할 수 있는지 확인합니다.
