# Feature: 빈 번역 찾기 기능 및 UI 개선 (v1.3)

## 개요

번역 관리 페이지의 사용성을 개선하고, 번역 실패 시 데이터 저장 방식을 변경하며, UI 버그를 수정합니다.

## 주요 변경 사항 (v1.3)

-   **'빈 번역 찾기' 예외 처리**: '빈 번역 찾기' 필터 사용 시, 결과가 하나도 없을 경우 화면을 비우는 대신 "빈 번역 기록이 없습니다." 라는 알림을 표시하도록 개선했습니다.

## 주요 변경 사항 (v1.2)

-   **렌더링 버그 수정**: '빈 번역 찾기' 또는 '검색' 기능 사용 시, 폴더 목록 데이터를 제대로 전달하지 못해 흰 화면이 표시되던 치명적인 버그를 수정했습니다.

## 주요 변경 사항 (v1.1)

-   **필터링 UX 개선**: 필터가 활성화되면, 결과가 포함된 폴더 및 URL 그룹이 자동으로 펼쳐져 바로 내용을 확인할 수 있도록 개선했습니다.
-   **UI 레이아웃 수정**: '전체 찾아 바꾸기' 버튼이 '폴더 생성' 영역을 침범하던 CSS 레이아웃 문제를 수정했습니다.
-   **이벤트 리스너 수정**: '빈 번역 찾기' 버튼이 간헐적으로 동작하지 않던 문제를 해결했습니다.

## 주요 변경 사항 (v1.0)

-   **'빈 번역 찾기' 기능 추가**: 번역 관리 페이지에 '빈 번역 찾기' 버튼을 추가하고, 관련 필터링 로직을 구현했습니다.
-   **번역 실패 시 저장 로직 변경**: 번역 최종 실패 시, 빈 문자열(`''`)로 서버에 저장하도록 로직을 변경했습니다.
-   **재시도 로직 수정**: 단일 항목 번역 실패 시 재시도 로직을 개선했습니다.

## 영향

-   **`server/static/admin.js`**:
    -   `toggleFindEmpty` 함수에 빈 번역 기록이 없을 경우에 대한 예외 처리 로직을 추가했습니다.
    -   폴더 목록(`allFolders`)을 전역 상태로 관리하도록 변경하고, `renderGroupedHistory` 함수의 렌더링 오류를 해결했습니다.
-   **`extension/content.js`**: `retryWithSentenceSplitting` 및 `handleFailure` 함수가 수정되었습니다.
-   **`server/templates/admin.html`**: '빈 번역 찾기' 버튼이 추가되었습니다.
-   **`server/static/admin.css`**: 컨트롤 바의 레이아웃을 수정했습니다.
