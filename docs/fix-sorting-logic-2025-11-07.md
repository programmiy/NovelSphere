# 정렬 로직 수정 (2025-11-07)

## 문제

`get_translations_by_folder` API의 정렬 로직이 `first_accessed_at` 기준으로 변경된 후, 이 값이 없는 기존 데이터들의 순서가 엉망이 되는 문제가 발생했습니다.

## 해결

`first_accessed_at` 타임스탬프가 없는 URL 그룹의 경우, 해당 그룹 내에서 가장 오래된 번역 데이터의 `timestamp`를 정렬 기준으로 사용하도록 로직을 수정했습니다.

이를 통해 모든 URL 그룹이 일관된 시간 순서로 정렬되도록 문제를 해결했습니다.
