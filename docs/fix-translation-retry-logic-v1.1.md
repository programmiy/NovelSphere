# Fix: 번역 재시도 로직 및 실패 처리 개선 (v1.1)

## 개요

번역 과정의 오류 처리 및 재시도 로직을 개선하여 안정성과 사용자 경험을 향상합니다.

## 변경 사항 (v1.1)

- **재시도 로직 개선**: 단일 항목 번역 실패 시, 불필요하게 묶음 단위를 줄이려 시도하던 비효율적인 로직을 수정했습니다. 이제 실패한 항목이 1개일 경우, 즉시 개별 항목 재시도(문장 분할 등) 로직으로 전환됩니다.

## 변경 사항 (v1.0)

- **분할 재시도 실패 처리**: `PROHIBITED_CONTENT` 등의 이유로 문장 분할 재시도 후에도 번역에 실패하는 경우, 기존의 `번역 실패: 분할 재시도 실패` 오류 메시지를 번역 결과에 표시하는 대신 빈 텍스트를 삽입합니다.
- **실패 상태 제목 표시**: 번역 실패가 발생한 경우, 해당 문서의 제목(`document.title`) 맨 앞에 `[번역 실패: 분할 재시도 실패 + 실패한 pid 위치:(해당 pid)]` 형식의 태그를 추가합니다. 이 변경된 제목은 번역 기록 저장 시 함께 저장되어, 사용자가 나중에 어떤 문서에서 오류가 발생했는지 쉽게 식별할 수 있도록 합니다.

## 영향

-   **`extension/content.js`**:
    -   `handleFailure` 함수: 실패한 항목이 1개일 때를 명시적으로 처리하여 불필요한 묶음 재시도를 방지하도록 수정했습니다.
    -   `retryWithSentenceSplitting` 함수의 `catch` 블록: 최종 실패 시 빈 텍스트를 표시하고 `document.title`을 변경하도록 수정했습니다.
