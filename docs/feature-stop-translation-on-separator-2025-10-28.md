# 구분선 이후 번역 중단 기능 추가

## 기능 요약

웹 페이지 본문에서 "——"와 같이 연속된 대시로만 이루어진 구분선이 나타나면, 그 이후의 모든 문단(PID)에 대한 번역을 자동으로 중단하는 기능을 추가했습니다.

## 도입 배경

많은 웹 소설 작가들이 본문 내용이 끝난 후 "——" 구분선을 넣고 그 뒤에 자신의 잡담이나 후기를 작성합니다. 이 부분은 번역할 필요가 없는 내용이지만, 기존 로직에서는 일본어가 포함되어 있다는 이유만으로 모두 번역을 시도하여 불필요한 API 비용이 발생했습니다. 또한, 원하지 않는 내용이 데이터베이스에 저장되는 문제도 있었습니다.

## 구현 내용

- **`extension/content.js`의 `translateAllPTags` 함수 수정**:
    1.  페이지 내 문단을 순차적으로 분석하는 로직에 `skipTranslation`이라는 플래그 변수를 추가했습니다.
    2.  문단 텍스트가 공백과 '—' 문자로만 구성된 경우, 이를 작가 후기의 시작을 알리는 구분선으로 간주합니다.
    3.  구분선이 감지되면 `skipTranslation` 플래그를 `true`로 설정하고, 해당 구분선 자체는 번역 대상에서 제외합니다.
    4.  이후의 모든 문단은 번역을 시도하기 전에 `skipTranslation` 플래그를 먼저 확인하게 됩니다. 플래그가 `true`이면 해당 문단은 일본어 포함 여부와 관계없이 번역 대상에서 즉시 제외됩니다.

## 기대 효과

- 작가 후기와 같은 불필요한 텍스트의 번역을 방지하여 API 호출 비용을 절감합니다.
- 데이터베이스에 순수 본문 내용만 저장되도록 하여 데이터의 질을 향상시킵니다.
