# 번역 기록 페이지 UI/UX 개선 (v1.0)

## 개요
번역 기록 페이지의 사용자 경험(UX) 및 사용자 인터페이스(UI)를 개선하기 위한 첫 번째 단계로, 디자인 테마, 레이아웃 구조, 핵심 기능 UI를 업데이트했습니다.

## 주요 변경 사항

### 1. 새로운 디자인 테마 및 스타일
- **Material Design 아이콘 도입:** Google Material Icons CDN을 추가하여 UI 요소에 현대적이고 일관된 아이콘을 적용했습니다.
- **라이트/다크 모드 지원:**
    - `history.html`의 `:root` 및 `body.dark-mode` CSS 변수를 확장하여 라이트/다크 모드 테마를 위한 색상 팔레트를 정의했습니다.
    - `controls-bar`에 다크 모드 토글 버튼을 추가하고, `history.js`에서 사용자 선호도를 `chrome.storage.local`에 저장하고 로드하는 기능을 구현했습니다.
    - 토글 버튼의 아이콘이 현재 모드에 따라 `dark_mode` 또는 `light_mode`로 변경되도록 했습니다.

### 2. 레이아웃 구조 개선
- **컨트롤 패널 재그룹화:** `controls-bar` 내의 검색/필터, 내보내기, 삭제, 다크 모드 토글 버튼들을 논리적으로 재그룹화하여 시각적 명확성과 사용성을 향상시켰습니다.

### 3. 핵심 기능 UI 개선
- **항목별 액션 버튼 숨기기/표시:** 각 번역 기록 항목(`history-item`)의 '수정', '복사', '삭제' 버튼을 기본적으로 숨기고, 해당 항목에 마우스를 올렸을 때만 나타나도록 CSS를 수정하여 시각적 혼잡도를 줄였습니다.
- **로딩 상태 표시 (스켈레톤 UI):**
    - `history-container`에 스켈레톤 UI 구조를 추가했습니다.
    - `history.html`에 스켈레톤 UI를 위한 CSS 스타일(`skeleton-group`, `skeleton-header`, `skeleton-item`, `pulse-bg` 애니메이션)을 정의했습니다.
    - `history.js`의 `loadAndRender` 함수에서 데이터를 불러오는 동안 스켈레톤 UI를 표시하고, 로딩이 완료되면 숨기도록 구현하여 사용자에게 시스템이 작동 중임을 시각적으로 알립니다.

## 향후 계획
- 테이블 형태의 목록 뷰로의 전환을 위한 추가적인 HTML/CSS/JS 수정
- 폴더/URL별 그룹화 아코디언 기능 개선
- 일괄 작업 기능의 시각적 피드백 강화
- 애니메이션 효과 추가 및 개선
