# Fix: `PROHIBITED_CONTENT` 오류 시 재시도 로직 개선 (2025-11-14)

## 설명

Gemini API로부터 `PROHIBITED_CONTENT` 오류를 수신했을 때의 재시도 로직을 개선했습니다.

기존에는 클라이언트(확장 프로그램) 측에서 번역 묶음(chunk)을 절반으로 나누어 재귀적으로 재시도하는 방식이었습니다. 이 방식은 특정 콘텐츠가 반복적으로 문제를 일으킬 경우 비효율적일 수 있었습니다.

새로운 로직은 서버 측에서 `PROHIBITED_CONTENT` 오류를 감지하면, 묶음 전체를 실패 처리하는 대신 **개별 항목 번역 모드**로 자동 전환합니다. 즉, 실패한 묶음에 포함된 모든 문장을 하나씩 개별적으로 번역 API에 다시 요청합니다.

## 주요 변경 사항

-   **서버 측 재시도 로직 구현 (`server/routers/translation.py`):**
    -   `/translate` 엔드포인트에서 `PROHIBITED_CONTENT` 오류를 감지하면, HTTP 500 오류를 즉시 반환하는 대신 내부적으로 개별 번역을 시도합니다.
    -   각 문장을 개별적으로 번역하며, 특정 문장이 또다시 차단되더라도 전체 작업이 중단되지 않고 해당 문장만 실패로 처리한 후 다음 문장으로 넘어갑니다.
    -   이를 통해 어떤 문장이 문제를 일으키는지 정확히 식별할 수 있으며, 성공적으로 번역된 나머지 부분의 결과를 정상적으로 반환하여 번역 성공률을 높였습니다.

## 기대 효과

-   콘텐츠 차단 오류에 대한 대응력이 향상되어 전체 번역 성공률이 증가합니다.
-   클라이언트 측의 복잡한 재시도 로직 부담을 줄이고, 서버에서 오류를 더 안정적으로 처리합니다.
