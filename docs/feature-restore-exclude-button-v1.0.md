# 기능 복구: 번역 제외 버튼 (v1.0)

## 변경 사항 요약
서버 기반 아키텍처로 리팩토링되면서 누락되었던 '번역 제외' 기능을 복구했습니다. 이제 사용자는 번역 기록 UI에서 특정 문장을 영구적으로 번역 대상에서 제외할 수 있습니다.

## 상세 내용

### 백엔드 (`server/server.py`)
1.  **`excluded_sentences` 테이블 추가**:
    *   데이터베이스(`translations.db`)에 `excluded_sentences`라는 새 테이블을 추가했습니다.
    *   이 테이블은 `url`과 `original` (원문)을 저장하여 어떤 페이지의 어떤 텍스트를 번역에서 제외할지 관리합니다.
    *   `UNIQUE(url, original)` 제약 조건을 추가하여 중복 저장을 방지합니다.

2.  **`/translations/exclude` 엔드포인트 추가**:
    *   `POST` 요청을 통해 제외할 번역 항목의 ID 목록을 받습니다.
    *   요청된 ID에 해당하는 항목을 `translations` 테이블에서 찾아 `url`과 `original` 정보를 `excluded_sentences` 테이블에 저장합니다.
    *   저장 후, 해당 항목을 `translations` 테이블에서 삭제하여 번역 기록에 더 이상 표시되지 않도록 합니다.

3.  **`/translate` 엔드포인트 수정**:
    *   번역 요청 시, 먼저 `excluded_sentences` 테이블을 확인합니다.
    *   요청된 텍스트가 해당 URL의 제외 목록에 포함되어 있으면, Gemini API로 보내지 않고 즉시 빈 문자열로 처리합니다.
    *   이를 통해 제외된 문장이 다시 번역되는 것을 방지합니다.

### 프론트엔드 (`extension/history.js`)
1.  **'제외' 버튼 UI 복구**:
    *   번역 기록의 각 항목(row)에 '제외' 버튼(🚫)을 다시 추가했습니다.

2.  **이벤트 핸들러 추가**:
    *   '제외' 버튼 클릭 시, 사용자에게 영구적으로 제외할 것인지 확인하는 대화상자를 표시합니다.
    *   사용자가 확인하면, 백엔드의 `/translations/exclude` 엔드포인트로 해당 번역 항목의 ID를 전송합니다.
    *   요청이 성공적으로 처리되면, 해당 항목을 UI에서 즉시 제거하고 사용자에게 알림(toast)을 표시합니다.

## 기대 효과
- 사용자는 원치 않는 문장이 반복적으로 번역되는 것을 방지하여 번역의 정확성과 편의성을 높일 수 있습니다.
- 서버 마이그레이션 과정에서 손실되었던 중요 기능을 복구하여 기존 사용자 경험을 유지합니다.
