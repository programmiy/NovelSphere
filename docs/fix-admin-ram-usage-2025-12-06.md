# 어드민 기능 메모리 사용량 최적화

## 변경 사항

- **파일:** `server/routers/admin.py`
- **함수:** `cleanup_pids`
- **내용:** `fetchall()`을 사용하여 모든 레코드를 메모리에 로드하는 대신, `INSERT ... SELECT`와 `DELETE` 구문을 사용하여 데이터베이스 내에서 직접 데이터를 처리하도록 로직을 변경했습니다.

## 변경 이유

- `cleanup_pids` 함수 실행 시, 대량의 데이터를 메모리에 한 번에 불러와 발생하는 과도한 RAM 사용 문제를 해결하기 위함입니다.
- 기존 방식은 `translations` 테이블에서 조건에 맞는 모든 레코드를 애플리케이션 메모리로 가져온 후 처리하여, 데이터 양에 따라 서버에 큰 부담을 줄 수 있었습니다.

## 영향

- 이제 대용량 데이터도 메모리 급증 없이 효율적으로 처리할 수 있습니다.
- 서버의 전반적인 안정성이 향상됩니다.
