# 버그 수정: 전체 선택 시 하위 체크박스 시각적 업데이트 강제 (v1.0)

## 변경 사항 요약
'전체 선택' 체크박스 클릭 시, 내부 항목들의 체크박스 상태가 내부적으로만 변경되고 시각적으로는 반영되지 않던 고질적인 버그를 해결했습니다. 하위 체크박스의 `.checked` 속성을 직접 설정하는 대신, 프로그래밍 방식으로 `.click()` 이벤트를 트리거하여 브라우저의 렌더링을 강제하는 방식으로 문제를 해결했습니다.

## 문제 원인
여러 차례의 수정에도 불구하고, 폴더의 '전체 선택' 체크박스를 클릭했을 때 하위 체크박스들이 시각적으로 업데이트되지 않는 문제가 지속되었습니다. 폴더 접힘(collapse) 문제나 DOM 요소 선택자(selector)의 문제는 아니었으며, `.checked` 속성을 직접 변경하는 것만으로는 브라우저의 시각적 렌더링이 안정적으로 트리거되지 않는 특수한 상황으로 판단됩니다.

## 상세 수정 내용

### `extension/history.js`
-   **`handleCheckboxChange` 함수 로직 변경**:
    -   폴더의 '전체 선택' 체크박스(`folder-select-all-checkbox`)가 변경되었을 때의 동작을 수정했습니다.
    -   기존에는 폴더 내의 모든 개별 아이템 체크박스(`.item-checkbox`)를 찾아 `.checked` 속성을 직접 설정했습니다.
    -   새로운 로직에서는, 폴더 내에 있는 각 URL 그룹의 '전체 선택' 체크박스(`.group-select-all-checkbox`)를 찾아, 상태가 일치하지 않을 경우 해당 체크박스의 `.click()` 메소드를 호출합니다.
    -   이 `.click()` 호출은 실제 사용자가 클릭한 것과 같은 효과를 내어, 해당 그룹의 '전체 선택' 이벤트 핸들러를 연쇄적으로 발동시킵니다. 이 연쇄 반응이 하위 모든 항목의 체크박스 상태를 안정적으로 변경하고, 시각적 업데이트를 보장합니다.
    -   또한, URL 그룹에 속하지 않은 항목이 있을 경우를 대비해 모든 개별 아이템 체크박스의 `.checked` 속성을 직접 설정하는 기존 로직도 안전장치(fallback)로 남겨두었습니다.

## 기대 효과
-   **고질적 버그 해결**: 어떤 상황에서든 '전체 선택' 기능이 시각적으로 명확하고 안정적으로 동작하게 되어, 사용자가 의도한 대로 모든 항목이 선택/해제되는 것을 직접 확인할 수 있습니다.
-   **이벤트 기반 로직 강화**: 속성을 직접 제어하는 방식에서 벗어나, 이벤트(click)를 통해 다른 이벤트 핸들러를 연쇄적으로 활성화하는 방식으로 변경하여 코드의 동작 방식을 더 견고하고 예측 가능하게 만들었습니다.
