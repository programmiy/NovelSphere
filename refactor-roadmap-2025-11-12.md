# 프로젝트 고도화 로드맵 v2

## 1. 의존성 관리 체계화 (완료)

- **[X] 내용:** `pip freeze > requirements.txt` 명령을 사용하여 현재 프로젝트의 모든 Python 의존성을 `requirements.txt` 파일에 명시적으로 기록하고, 향후 모든 패키지 변경 사항을 이 파일을 통해 관리합니다.
- **[X] 기대 효과:** 개발 환경의 일관성을 보장하고, 새로운 개발자가 프로젝트에 쉽게 참여할 수 있도록 합니다. 배포 시 정확한 패키지 버전을 설치하여 예기치 않은 오류를 방지합니다.

## 2. 설정 관리 개선 (완료)

- **[X] 내용:** `Pydantic`의 `BaseSettings`를 도입하여 환경 변수 기반의 설정 관리를 구현합니다. `settings.json`, `api_key.txt` 등 여러 파일에 흩어져 있는 설정들을 중앙화하고, 개발/프로덕션 환경별로 다른 설정을 쉽게 주입할 수 있도록 개선합니다.
- **[X] 기대 효과:** 설정의 중앙화로 유지보수성이 향상되고, 민감한 정보(API 키 등)를 코드와 분리하여 보안을 강화합니다.

## 3. API 문서 자동화 및 고도화 (완료)

- **내용:** FastAPI의 자동 문서 생성 기능(Swagger UI, ReDoc)을 적극적으로 활용합니다. 각 API 엔드포인트에 `description`, `summary`, `response_model`, `tags` 등을 명확하게 추가하여, 별도의 문서 없이도 API의 기능과 사용법을 쉽게 파악할 수 있도록 합니다.
- **기대 효과:** API 명세가 항상 최신 상태로 유지되며, 프론트엔드 개발 또는 API 연동 시 협업 효율성이 증대됩니다.

## 4. 테스트 커버리지 확대 (완료)

- **내용:**
    - **통합 테스트:** Celery를 사용한 비동기 작업의 성공/실패 시나리오에 대한 통합 테스트를 작성합니다.
    - **API 테스트:** 주요 API 엔드포인트에 대한 테스트 케이스를 보강하여, 다양한 입력값과 엣지 케이스를 검증합니다.
    - **(선택) E2E 테스트:** `Playwright` 또는 `Selenium`을 도입하여 사용자 관점의 End-to-End 테스트를 작성, 회원가입, 로그인, 글쓰기 등 핵심 사용자 시나리오의 안정성을 확보합니다.
- **기대 효과:** 코드 변경에 대한 신뢰도를 높이고, 배포 전 잠재적인 버그를 사전에 발견하여 시스템 안정성을 향상시킵니다.

## 5. Docker를 이용한 컨테이너화 (진행 예정)

- **내용:** `Dockerfile`과 `docker-compose.yml`을 작성하여 FastAPI 애플리케이션, Celery 워커, Redis를 각각의 컨테이너로 실행할 수 있도록 환경을 구성합니다.
- **기대 효과:** 개발 환경 구성을 단순화하고, "내 컴퓨터에서는 되는데..."와 같은 문제를 원천적으로 차단합니다. 배포 과정을 표준화하고 확장성을 확보합니다.
